## 摘要
我们提出了MoVerse，一个实时视频世界模型，它能够从单个窄视场图像创建可交互导航的场景。这种设置具有挑战性，因为输入仅观测到环境的一小部分，而交互式漫游需要一个完整的周围世界、持久的几何结构、可控的相机运动以及时间上连贯的高保真观测。MoVerse通过将世界构建与观测渲染分离来解决这个问题。它首先通过拓扑感知扩散将输入扩展为重力对齐的360°全景图，在3D推理之前闭合缺失的视场。然后，利用全景几何感知残差预测将全景图提升为持久的3D高斯支架，生成密集且可直接渲染的空间记忆。最后，一个高斯条件视频渲染器将沿用户指定相机轨迹的支架渲染结果转换为逼真的视频。为了使该渲染器适用于交互，我们训练了一个双向扩散教师模型用于高质量条件渲染，并将其蒸馏为因果自回归学生模型以实现有界延迟流式传输。这种设计结合了显式3D表示的可控性和长程一致性以及生成式视频模型的感知质量。MoVerse在单个NVIDIA RTX 4090 GPU上支持8 FPS的实时场景漫游，展示了具有交互式视频输出的单图像世界创建的一条实用路径。

## Abstract
We present MoVerse, a real-time video world model that creates an interactively navigable scene from a single narrow-field-of-view image. This setting is challenging because the input observes only a small fraction of the environment, while interactive roaming requires a complete surrounding world, persistent geometry, controllable camera motion, and temporally coherent high-fidelity observations. MoVerse addresses this problem by separating world construction from observation rendering. It first expands the input into a gravity-aligned 360$^\circ$ panorama with topology-aware diffusion, closing the missing field of view before 3D reasoning. It then lifts the panorama into a persistent 3D Gaussian scaffold using panoramic geometry-aware residual prediction, yielding a dense and directly renderable spatial memory. Finally, a Gaussian-conditioned video renderer translates scaffold renderings along user-specified camera trajectories into photorealistic video. To make this renderer practical for interaction, we train a bidirectional diffusion teacher for high-quality conditional rendering and distill it into a causal autoregressive student for bounded-latency streaming. This design combines the controllability and long-range consistency of explicit 3D representations with the perceptual quality of generative video models. MoVerse supports real-time scene roaming at 8~FPS on a single NVIDIA RTX~4090 GPU, demonstrating a practical path toward single-image world creation with interactive video output.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：从单张窄视场（NFOV）图像创建可交互漫游的完整3D场景。输入仅观察到环境的一小部分，而交互式漫游需要：
  - 完整的周围世界（360°环境）
  - 持久的几何结构（空间记忆）
  - 可控的相机运动
  - 时间上连贯的高保真观测
- **现有方法的不足**：
  - 显式3D方法（如点云、网格、3D高斯）可构建持久场景资产并支持实时渲染，但从单张图像直接重建时，大部分场景需推断，易产生空洞、漂浮物、深度错误，且感知质量有限。
  - 隐式视频/世界模型（如生成式视频模型）可产生视觉质量高的视频，但长程几何不稳定，当用户长轨迹运动或回访时，场景可能漂移。
  - 混合系统使用显式几何作为生成渲染的条件，但锚点的密度与渲染器的实时性存在矛盾：稀疏锚点空间引导弱，而强生成渲染器成本高或不适用于交互。
- **论文目标**：设计一种分离世界构建与观测渲染的实用框架，使显式条件密集、全景、可直接渲染且可复用，同时视频模型被蒸馏为因果渲染器用于流式交互。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
将过程分为**离线世界构建**和**在线观测渲染**两个阶段：
- **Stage I & II（离线）**：从单张NFOV图像生成重力对齐的360°全景图，再将其转换为持久的3D高斯支架（Gaussian Scaffold）。该支架是密集、可直接渲染的空间记忆。
- **Stage III（在线）**：沿用户指定相机轨迹渲染3D高斯支架得到RGB条件视频流，然后通过高斯条件视频渲染器（蒸馏后的因果自回归学生模型）将其转换为高保真视频，实现有界延迟流式交互。

### 关键技术细节

#### Stage I：全景生成（NFOV → 全景图）
- **输入**：单张NFOV图像（未标定姿态）。
- **步骤**：
  1. **重力对齐**：使用可微自动调平模块，预测密集二维对应场，通过可微软argmin求解器提取刚性球面变换，将输入投影到重力对齐的标准观察空间（消除俯仰和滚转）。
  2. **拓扑感知扩散补全**：在标准空间中进行掩码潜在扩散补全。采用：
     - **循环潜在编码**：使VAE的水平卷积填充环绕方位轴，保持周期性。
     - **移位等变生成**：对扩散输入进行水平循环移位，确保生成结果对水平移位一致，消除人工接缝。
  3. **训练数据**：自建Horizon360数据集，由重力对齐的室内外全景图组成。从全景图中采样各种姿态的窄视角作为输入，目标为居中的全景图（输入视点居中），便于补全。

#### Stage II：全景高斯场景生成（全景图 → 3D高斯支架）
- **输入**：重力对齐的360°全景图及其估计深度。
- **步骤**：
  1. **全景高斯初始化**：根据深度球面反投影将像素点映射为高斯中心。考虑纬度对球面面积的影响，初始尺度与 \( D_k \cos \phi_k \) 成正比，避免高纬度退化。
  2. **深度引导残差预测**：基于SHARP的深度引导残差预测思想，但适应全景几何。提取多尺度图像特征及显式特征（RGB + 逆深度），用一个轻量级残差预测头预测高斯基元属性的残差：角度偏移、逆深度偏移、尺度残差、旋转残差、颜色残差、不透明度残差。
  3. **属性组合**：在角度-逆深度空间应用残差，再通过球面反投影得到最终的3D高斯中心。其他属性也以残差方式更新，保证初始化物理合理。
- **训练目标**：在HM3D数据集上使用可微渲染，包括目标视图重建损失（L1、LPIPS、SSIM）、源视图重建损失、逆深度损失、位置残差正则化。

#### Stage III：高斯条件视频渲染（条件RGB流 → 高保真视频）
- **步骤**：
  1. **支架渲染**：给定用户轨迹，用3D高斯溅射渲染得到条件视频流 \(\hat{V}_{1:T}\)。
  2. **双向扩散教师模型**：初始化为Wan2.1-T2V-1.3B，输入为目标视频VAE潜变量和条件视频VAE潜变量的通道拼接，共享旋转位置编码（Shared-RoPE），训练流匹配目标。教师可双向关注整个片段，用于高质量条件渲染。
  3. **因果自回归学生模型**：蒸馏教师模型，用于因果低延迟流式生成。学生模型每次生成一个潜在块（对应4像素帧），条件于当前条件帧块，并通过局部KV缓存（MemRoPE风格：一个sink帧、一个长期EMA记忆token、三个局部帧滑动窗口）维护短时时间连续性。使用两步去噪调度和TAEHV解码器加速。
- **训练数据混合**：三类配对数据，按表示驱动递进：
  - 真实视频 + 基于单帧深度重投影的轨迹对齐几何代理（训练修复基本遮挡）。
  - 从真实视频关键帧构建的3D高斯溅射渲染（学习高斯伪影修复）。
  - 全景→高斯支架渲染（匹配MoVerse部署接口），来源包括UE渲染全景、LayerPano3D、Matterport3D、Polyhaven HDRIs。
