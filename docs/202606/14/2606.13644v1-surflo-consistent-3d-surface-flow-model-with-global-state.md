## 摘要
几何形状对视角具有不变性，这使得任何图像集合都是对单个3D状态的冗余编码。现有的前馈重建模型未能利用这一点：逐视角方法生成重叠且未对齐的点云，其数量随输入数量线性增长；而全局隐变量方法则局限于固定的低分辨率输出。我们提出Surflo，它将任意数量的未标定RGB视图压缩为K个隐变量token——一个全局状态，并通过流匹配将噪声点独立地传输到表面上，从而解码出朝向一致的3D表面点。这使得输出不受任何固定网格或token数量的限制：相同的隐变量在单次前向传播中即可生成从几千到一百万个点。为了抑制独立逐点解码固有的局部不一致性，我们在推理时引入一个引导项，通过在ODE积分过程中注入光度梯度来关联相邻点。Surflo在表面指标上匹配或超越前馈基线，运行速度比需要数百个视图的基于优化方法快一个数量级，并且是唯一结合全局隐变量与任意分辨率解码的前馈方法。

## Abstract
Geometry is invariant to viewpoint, which makes any collection of images a redundant encoding of a single 3D state. Existing feed-forward reconstruction models fail to exploit this: per-view methods emit overlapping, unaligned pointmaps that grow linearly with input count, while global-latent methods commit to a fixed, low-resolution output. We introduce Surflo, which compresses a variable number of unposed RGB views into K latent tokens-one global state-and decodes oriented 3D surface points by independently transporting them from noise onto the surface via flow matching. This frees the output from any fixed grid or token budget: the same latent yields from a few thousand to a million points in a single forward pass. To suppress the local inconsistencies inherent to independent per-point decoding, an inference-time guidance term correlates nearby points by injecting a photometric gradient during ODE integration. Surflo matches or surpasses feed-forward baselines on surface metrics, runs an order of magnitude faster than optimization-based methods that require hundreds of views, and is the only feed-forward approach to combine a global latent with arbitrary-resolution decoding.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现有前馈式3D重建方法未能利用“几何对视角不变”这一性质。逐视角方法（如VGGT、DUSt3R）输出重叠、未对齐的点云，token数量随输入视图数线性增长，难以融合为全局一致网格；全局隐变量方法（如NOVA3R）虽然压缩多视图为单一表示，但输出分辨率固定、不能灵活扩展。
- **背景意义**：在许多应用（如机器人real2sim、智能体导航）中，最终目标是获取全局一致的场景几何，而非逐视角的深度图或点云。因此需要一种既能压缩视图冗余、又能按需采样任意分辨率表面点的前馈模型。
- **本文目标**：提出Surflo，将任意数量未位姿RGB视图压缩为固定大小隐变量，并通过流匹配从噪声中独立解码出任意数量的朝向表面点，同时通过推理时引导机制保证全局一致性。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：三个关键组件协同工作：
  1. **编码器**：将$N$张未位姿RGB视图压缩为固定大小的全局隐变量$z \in \mathbb{R}^{K \times D}$（$K=128$）。
  2. **解码器**：基于流匹配，对每个查询点$x_t \in \mathbb{R}^3 \times \mathbb{S}^2$独立预测速度场$v_\theta(x_t, t | z)$，将其从噪声分布传输到表面分布。支持任意数量的点（几千到一百万）在单次前向传播中并行解码。
  3. **引导机制**（推理时）：在ODE积分后期（$t \ge 0.95$），利用可微渲染损失（光度+深度）的梯度修正速度场，使独立点之间相互通信，抑制局部不一致性和离群点。
- **编码器细节**：
  - 使用冻结的VGGT-1B骨干网络提取每视图的patch token（4层）和相机token。
  - 用3D高斯傅里叶特征编码patch中心的三维坐标，加到对应token上。
  - 通过Perceiver风格的交叉注意力将$N \times 4N_p$个token压缩为$K$个隐token（$K=128$）。
- **解码器细节**：
  - 12层Transformer，前6层包含交叉注意力（与隐变量）和MLP，后6层仅MLP；均使用AdaLN条件化（时间+相机token）。
  - 训练目标为流匹配损失：$\mathcal{L}_{\text{FM}} = \mathbb{E}[ \| v_\theta(x_t, t|z) - (x_1 - x_0) \|_2^2 ]$。
  - 源分布：使用混合高斯（以VGGT点云样本为中心加噪），而非标准正态，以集中在几何附近。
- **引导机制**：
  - 在$t \ge 0.95$时，预测当前点的目标$\hat{x}_1$，将其视为小高斯，通过RaDe-GS光栅化渲染到输入视图，计算光度损失$\mathcal{L}_{\text{render}}$，并执行$M=32$步梯度下降修正目标，得到引导速度$v_g$，用于实际ODE步进。
  - 可选的单目深度引导（使用DepthAnything-3）进一步锐化几何。
- **最终输出**：导向高斯点云可通过Delaunay三角化转换为网格。

## 6. 主要结论与发现

- **主要结论**：Surflo在表面指标上匹配或超越所有前馈基线（无论是逐视角还是隐变量），且速度比任何优化方法快至少一个数量级。
- **关键发现**：
  - 全局隐变量+任意分辨率解码的组合是可行的，并且优于固定分辨率的隐变量方法（NOVA3R）。
  - 逐点独立解码+推理时引导，能有效抑制局部不一致，产生全局一致的表面。
  - 基于渲染的引导显著提升细节质量，尤其是在稀疏视图和遮挡区域。
  - 模型对视图数量鲁棒：从2视图到32视图，性能稳定提升，且解码成本不变（隐变量相同）。
  - 生成的点云可直接通过Delaunay三角化转为高质量网格，无需复杂后处理。

## 7. 优点

- **方法创新**：首次将全局隐变量与任意分辨率流匹配解码结合，支持从千到百万点的灵活采样，是真正的“压缩-查询”范式。
- **效率极高**：前馈方式，推理时间远低于优化方法；编码一次隐变量后可重复用于不同分辨率。
- **架构简洁**：逐点独立处理，无自注意力，易于并行；利用冻结的VGGT骨干，减少训练成本。
- **引导机制独创**：用可微渲染损失作为“通信”信号，在不增加训练复杂度的情况下提升一致性。
- **附加大数据集贡献**：构建并公开带水密网格的DL3DV版本（约10.5k场景），可供后续场景级表面学习使用。
- **实验扎实**：覆盖8个基准，多视图数量变化，消融完整，结果客观。

## 8. 不足与局限

- **对骨干网络的依赖**：Surflo依赖VGGT生成的特征和初始点云；当VGGT在极少视图、极大基线或特殊场景下失效时，模型性能会下降。
- **引导成本**：虽然整体推理很快，但引导步骤（32次内循环更新）增加的负担在实时应用场景中可能仍嫌高。
- **真值质量限制**：训练监督来自Gaussian Wrapping生成的表面，在透明物体、无纹理区域等挑战场景下可能不完美，从而限制了模型的极限能力。
- **不支持外观**：当前模型仅输出几何（位置和法线），不预测视角相关辐射度，无法直接用于新颖视图合成或渲染。未来可扩展为同时预测颜色。
- **实验覆盖仍有空间**：未测试极端稀疏（1视图）或极密（>100视图）的情况；未与最新的大规模重建模型（如Fast3R、MapAnything等）对比（因论文截止时间或这些方法尚未发布）；消融中未测试编码器替换（如其他骨干）。
- **潜在偏差**：训练数据主要来自DL3DV（互联网视频），可能偏向某些场景类型（室内、室外建筑），泛化到极端环境（高反射、水下等）未验证。
