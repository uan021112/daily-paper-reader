## 摘要
通用视觉-语言-动作系统需要以对象为中心的3D证据和可复用的操作经验来规划可靠的机器人轨迹。GeneralVLA提供了将语言和RGB-D观测转换为3D末端执行器路径的分层接口，但仍存在两个瓶颈。首先，单目SAM3D风格的对象重建可能产生姿态和未见几何的幻觉，而当存在校准的多视图观测时，操作受益于稳定的对象形状。其次，原始KnowledgeBank主要检索语义相似的片段并附加新知识，这使得控制记忆质量、冲突、置信度和几何相关性变得困难。为解决第一个挑战，我们引入GeoFuse-MV3D，一种几何先验引导的MV-SAM3D重建分支，它利用输入视图掩码验证外部几何线索，应用软视觉外壳支持，执行轴方向细化，并在保留外观的同时融合几何信息。为解决第二个挑战，我们将KnowledgeBank升级为一个受控的长期记忆系统，具有明确的质量、置信度、生命周期、验证器和冲突元数据，以及面向精度的检索。最后，我们在GSO-30上评估重建分支，在Terminal-Bench 2.0和SWE-Bench Verified上评估记忆模块；GeoFuse-MV3D相比MV-SAM3D基线，将CD和LPIPS降低了2.20%和2.02%，同时将PSNR和SSIM提高了2.36%和1.03%；KnowledgeBank在Terminal-Bench SR上相比ReasoningBank改进4.53%，在SWE-Bench解决率上改进3.73%，同时分别降低AS 4.95%和5.65%。代码：https://github.com/AIGeeksGroup/GeneralVLA-2。网站：https://aigeeksgroup.github.io/GeneralVLA-2。

## Abstract
Generalist vision-language-action systems need object-centric 3D evidence and reusable manipulation experience to plan reliable robot trajectories. GeneralVLA provides a hierarchical interface for converting language and RGB-D observations into 3D end-effector paths, but two bottlenecks remain. First, monocular SAM3D-style object reconstruction can hallucinate pose and unseen geometry, while manipulation benefits from stable object shape when calibrated multi-view observations are available. Second, the original KnowledgeBank mainly retrieves semantically similar snippets and appends new knowledge, which makes it difficult to control memory quality, conflicts, confidence, and geometric relevance. To address the first challenge, we introduce GeoFuse-MV3D, a geometry-prior-guided MV-SAM3D reconstruction branch that verifies external geometry cues with input-view masks, applies soft visual-hull support, performs axis-wise refinement, and fuses only geometry while preserving appearance. To address the second challenge, we upgrade KnowledgeBank into a governed long-term memory system with explicit quality, confidence, lifecycle, verifier, and conflict metadata, together with precision-oriented retrieval. Finally, we evaluate the reconstruction branch on GSO-30 and the memory module on Terminal-Bench 2.0 and SWE-Bench Verified; GeoFuse-MV3D improves over the MV-SAM3D baseline by reducing CD and LPIPS by 2.20% and 2.02% while increasing PSNR and SSIM by 2.36% and 1.03%, and KnowledgeBank improves over ReasoningBank by 4.53% on Terminal-Bench SR and 3.73% on SWE-Bench resolve rate, while reducing AS by 4.95% and 5.65%, respectively. *   **核心问题**：尽管GeneralVLA架构有效，但其在机器人规划的两个关键输入证据上存在明显短板：
    1.  **物体3D重建不准确**：依赖单目SAM3D风格重建，容易产生物体姿态和背面的几何幻觉，这对需要精确抓取和避障的机器人操作任务有害。
    2.  **经验复用不可靠**：原始知识库（KnowledgeBank）仅基于语义相似性检索记忆片段，无法保证记忆的质量、时效性、准确性以及与当前场景的几何相关性。
*   **整体含义**：为了解决这两个瓶颈，论文提出了GeneralVLA-2。核心思想是在保持原有分层架构不变的前提下，**增强规划器（3DAgent）所依赖的两个核心证据来源**：更可靠的当前场景物体几何信息（通过GeoFuse-MV3D）和更可信的长期操作经验（通过受控知识库）。目标是让机器人规划更稳健、更智能。

#### 2. 论文提出的方法论

论文提出了两大核心贡献模块，它们分别作用于感知和记忆层，旨在提升3DAgent的规划质量。

*   **2.1 GeoFuse-MV3D：几何感知的多视角物体重建分支**
    *   **核心思想**：利用可获得的校准多视角观测，以保守的方式修正MV-SAM3D（多视角SAM3D）的几何缺陷。它不是直接替代MV-SAM3D，而是在其基础上引入外部几何先验和输入图像约束进行精细校正。
    *   **关键技术细节**：
        1.  **双源几何校正**：
            *   **源A（几何先验主源）**：引入外部几何先验（论文中实例化为VGGT），并应用轻量级的外观仿射校准，以保持颜色纹理一致性。
            *   **源B（无先验轴补偿）**：不依赖外部先验，直接从输入视角的掩码和位姿中，对高斯中心进行独立的轴向缩放和平移优化，以改善对输入图像的遮挡一致性。
        2.  **保守融合策略**：
            *   **软视觉外壳（Soft Visual Hull）**：计算3D高斯点在输入视图上的投影与输入掩码的匹配度。对匹配度低的点，不直接删除，而是采用“软收缩”策略，将其向内（向物体中心）小幅缩小，避免产生空洞。
            *   **外观保护性融合**：仅在几何坐标（高斯中心、网格顶点）层面融合来自源A和源B的信息，而颜色、不透明度、缩放、旋转和球谐函数（SH）外观等非几何属性则完全保留来自源A（更可靠）的结果。这确保了视觉质量的稳定性。
        3.  **几何融合**：当两个源的高斯点索引兼容时，采用置信度加权的残差融合方式，对几何坐标进行混合，`p_out = (1-α)p_A + α*p_B`，其中α根据两个源的掩码一致性得分动态调整。

*   **2.2 Governed KnowledgeBank：受控的长期记忆系统**
    *   **核心思想**：将知识库从简单的“文本缓存”升级为一个带有精细元数据管理的“受控记忆系统”。每个记忆记录不仅有内容，还有关于其本身质量、来源、状态和适用性的完整档案。
    *   **关键技术细节**：
        1.  **结构化记忆记录**：每个记忆`m`是一个结构化元组，包含：查询`q`、内容`c`、类型`y`（如过程提示、失败约束）、生命周期状态`z`（暂定、活跃、摘要、归档）、置信度`κ`、质量分`R`、冲突/取代链接`L`、验证器元数据`v`。
        2.  **验证器驱动的准入机制**：在执行任务后，使用“LLM-as-Verifier”对从轨迹中提取的候选知识进行多准则评分（如任务完成度、空间一致性、碰撞安全性、可泛化性等）。只有评分足够高的候选知识才能被准入。
        3.  **面向精度的检索**：检索评分函数`S`不仅包含文本相似性`r_text`，还融合了记忆的置信度`κ`、成功历史、时效性、使用频率，并减去冲突和过时惩罚。
        4.  **完整生命周期管理**：设有合并、替换、丢弃、摘要和归档等治理机制，在固定的活跃记忆预算下，确保知识库的质量、一致性并抑制冗余。
