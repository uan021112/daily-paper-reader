## 摘要
本文介绍了EPS3D，一种用于开放词汇三维全景分割的新型端到端前馈框架。与依赖额外预处理的现有方法不同，我们设计了一种端到端架构，并采用基于蒸馏的训练策略在多样化的三维场景上从多视图图像预测三维感知的语义和实例特征，从而提升三维一致性并避免误差累积。我们进一步提出了一个相互增强模块，以强制执行固有的语义-实例一致性。通过在实例内对齐语义（Ins2Sem）以及利用语义指导细化实例特征（Sem2Ins），我们实现了更连贯的三维场景理解。最终，EPS3D在两个基准测试上以高效率（例如每场景1秒）超越了SOTA基线（例如在Replica上语义mIoU提升13%），支持机器人操作和三维场景编辑等任务。

## Abstract
This paper introduces EPS3D, a new end-to-end feed-forward framework for open-vocabulary 3D panoptic segmentation. Unlike existing methods relying on additional preprocessing, we design an end-to-end architecture, with a distillation-based training strategy on diverse 3D scenes to predict 3D-aware semantic and instance features from multi-view images, improving 3D consistency and avoiding error accumulation. We further propose a mutual enhancement module to enforce inherent semantic-instance consistency. By aligning semantics within instances (Ins2Sem) and refining instance features with semantic guidance (Sem2Ins), we achieve more coherent 3D scene understanding. Ultimately, EPS3D outperforms SOTA baselines on two benchmarks (e.g., +13% mIoU for semantics on Replica) with high efficiency (e.g., 1s per scene), supporting tasks like robotic manipulation and 3D scene editing.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：开放词汇三维全景分割（OV3DPS）要求同时预测场景中任意类别的语义标签和实例身份，并保证三维视图间的一致性。现有方法多采用“两阶段”流程：先利用2D基础模型（如CLIP、SAM）逐视图提取特征，再通过3D网络融合。这种做法存在两大缺陷：一是2D特征存在视图间不一致（语义和实例ID不匹配），导致后续融合产生误差累积；二是依赖额外的预处理（如逐场景优化3D高斯），耗时且缺乏场景级鲁棒性。
- **整体含义**：本文试图实现首个**端到端前馈**的三维全景分割框架，直接从未标定的多视图图像中预测统一的三维全景高斯表示，同时兼顾语义和实例预测，消除误差累积，提升效率和一致性。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：设计一种端到端架构，通过蒸馏2D基础模型的知识来训练，直接学习3D感知的语义和实例特征；并引入**语义-实例互增强模块**，利用两者的互补性促进一致性。
- **关键技术细节**：
  - **表示**：采用**3D高斯泼溅（3D-GS）** 作为统一表示，每个高斯包含几何/外观参数（中心、不透明度、旋转、尺度、球谐系数）以及语义特征（512维CLIP对齐）和实例特征（32维）。
  - **网络架构**：基于VGGT的几何变换器（L层交替注意力）提取3D感知token，再通过DPT解码器生成深度图（投影为高斯中心）和其他几何属性；同时使用两个额外的DPT头直接预测语义特征和实例特征（端到端，无需2D预处理）。
  - **训练损失**：
    - RGB渲染损失（L1）。
    - 语义损失（Lsem）：渲染的语义特征与2D LSeg教师特征之间的余弦相似度。
    - 实例对比损失（Lins）：基于SAM提供的2D实例掩码，在渲染的实例特征图上使用InfoNCE损失，实现排列无关的对比学习。
  - **语义-实例互增强模块**：
    - **Sem2Ins**：将初始实例特征与语义特征融合（通过投影+拼接+融合层），得到语义增强的实例特征，使实例特征更精确。
    - **Ins2Sem**：对每批随机选出的M个高斯，基于实例特征相似度选出K个邻居（视为同一实例），强制这些邻居的语义特征一致（LIns2Sem损失），增强语义边界。
  - **推理**：语义：计算渲染特征与文本原型（CLIP编码）的余弦相似度取最大；实例：对3D实例特征进行HDBSCAN聚类得到原型，再计算最近邻。
