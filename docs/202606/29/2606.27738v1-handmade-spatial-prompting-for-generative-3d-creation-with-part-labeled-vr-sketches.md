## 摘要
文本到3D的生成降低了3D内容创作的门槛，但仅靠文本在指定空间意图方面是弱界面：部件应该放在哪里，它们如何关联，以及一个物体如何在3D中组织。我们提出了HandMade，一种结合虚拟现实3D草图和语言的工作流程，用于开放领域的3D资产生成。HandMade将粗糙的、带有部件标签的3D草图不是视为直接重建的不完整几何体，而是作为现有生成模型的空间提示。它将分割的VR笔画转换为多视图部件引导和结构化提示，允许用户通过3D草图指定对象布局和部件关系，同时使用语言指定身份、材质、风格和局部细节。技术评估显示，在20个不同示例上，HandMade比仅文本和基于草图的基线更好地保留了用户创作的空间支架。一项包含八名参与者的用户研究描述了用户在初始创作和后续修订中如何利用3D草图进行空间布局，以及如何利用语言指定身份、材质和细节。HandMade贡献了一种交互范式和从界面到生成的管线，用于空间引导的3D创作。

## Abstract
Text-to-3D generation lowers the barrier to 3D content creation, but text alone is a weak interface for specifying spatial intent: where parts should be placed, how they relate, and how an object should be organized in 3D. We present HandMade, a workflow that combines VR 3D sketching and language for open-domain 3D asset generation. HandMade treats coarse, part-labeled 3D sketches not as incomplete geometry to reconstruct directly, but as spatial prompts for existing generative models. It converts segmented VR strokes into multi-view part guidance and structured prompts, allowing users to specify object layout and part relationships through 3D sketching while using language for identity, material, style, and local details. A technical evaluation shows that HandMade better preserves user-authored spatial scaffolds than text-only and sketch-based baselines on 20 varied examples. A user study with eight participants characterizes how users make use of 3D sketching for spatial layout and language for identity, materials, and details across initial authoring and subsequent revision. HandMade contributes an interaction paradigm and interface-to-generation pipeline for spatially guided 3D creation.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：文本作为文本到3D生成的主要接口，在表达空间意图方面非常薄弱。用户难以通过文本精确描述部件的位置、相对大小、连接关系和三维组织结构。
- **背景**：现有文本到3D方法（如DreamFusion、Magic3D）降低了3D生成门槛，但用户仍需要大量提示工程来指定空间布局。图像到3D方法（如Hunyuan3D-2mv）则假设用户已有较完整的视觉参考，不适合早期构思阶段。草图建模（如Teddy）显示了徒手绘制的表达能力，但VR草图通常是稀疏、不完整、有噪声的，不适合直接作为精确几何输入。
- **整体含义**：本文提出HandMade工作流，将粗糙的、带部件标签的VR 3D草图视为空间提示（spatial prompts），而不是待直接重建的不完整几何体。通过结合3D草图（提供空间布局）和自然语言（提供身份、材质、风格等语义），实现了开放领域的3D资产生成，弥补了文本和纯草图各自的不足。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **空间和语义分工**：3D草图承载部件位置、朝向、比例和空间关系；语言承载对象身份、材质、风格和局部细节。
- **松耦合管线**：将粗糙的、带部分标签的VR笔画转化为多视图部分引导（multi-view part guidance），并结合结构化文本提示，驱动现成的多视图图像生成模型和图像到3D重建模型，无需训练专门的草图到3D模型。

### 关键技术细节
1. **输入表示**：
   - 用户在VR中通过手部跟踪绘制3D笔画，系统自动为每个部件通道分配不同颜色。
   - 用户为每个部件提供简短名称和可选描述，以及整体对象描述。
   - 未标记的辅助笔画（scaffold strokes）用于对齐形状，但不传递到生成阶段。

2. **多视图部分引导生成**：
   - 将整个3D草图归一化到单位立方体。
   - 从四个固定视角（方位角0°、90°、180°、270°，俯仰0°）渲染带颜色编码的部件通道，拼接成一张2×2网格图像（多视图部分引导图）。
   - 该图编码了近似部件形状、位置和跨视图对应关系，但不暗示真实外观。

3. **结构化提示构建**：
   - 提示包含三部分：输出规格（要求生成干净的前/右/后/左多视图）→颜色到部件名称的映射→用户提供的对象描述和部件描述。
   - 明确告知生成模型：引导图来自徒手VR输入，是粗糙、有噪声的空间表达，文本描述优先决定身份和外观。

4. **多视图图像合成**：
   - 使用ChatGPT Images 2.0（图像编辑接口）结合多视图部分引导图和结构化提示，生成一张2×2多视图对象图像。
   - 程序化地分割为四个独立视图（前、左、后、右），去除白色背景。

5. **3D形状生成**：
   - 将四个视图输入Hunyuan3D-2mv（前馈多视图图像到3D模型），生成未纹理网格。
   - 使用二次误差度量简化网格至10万面，再通过Hunyuan3D-Paint生成最终纹理网格（glTF格式）。

6. **修订与编辑**：
   - 用户可：修改3D草图→修改文本描述→重新运行图像生成或3D重建。
   - 支持三种修订路径：调整布局、调整语义、探索随机变化。

## 3. 实验设计

### 数据集/场景
- **技术评估**：20个开放域设计，包括8个用户研究样例和2个试点会话样例。涵盖建筑、交通工具、家具、游乐结构、生物形态、抽象物体等。
- **用户研究**：8名参与者（24-41岁，M=31；3人有绘画/CAD/3D建模经验，1人日常使用VR），每人设计一个想象的3D对象。

### Benchmark
- 使用标准化单位边界框空间中的几何对齐指标：
  - **Sketch→Mesh距离**：草图点到网格采样的最近邻距离均值（测量生成物体覆盖用户草图的程度）。
  - **Mesh→Sketch距离**：网格采样点到草图的最近邻距离均值（测量生成表面靠近笔画的比率）。
  - **Chamfer距离**：两者平均值。
  - **Recall@0.05**：草图点最近距离<0.05的比率。
  - **Precision@0.05**：网格点最近距离<0.05的比率。
  - **F-score@0.05**：Recall和Precision的调和平均。

### 对比方法
- **HandMade**（完整流程）
- **Text-only**：去掉多视图部分引导图，仅用文本描述（消融）。
- **Single-view**：仅用单视图生成图像（消融）。
- **Instant3dit**：多视图修补/编辑基线，将3D草图投影到2×2布局并修补重建。
- **SketchDream**：草图+文本的3D生成基线，从前投影推导RGBA掩码和深度图。
- **LAS-Diffusion**：2D草图驱动的SDF扩散模型（无文本）。
- **MeshPad**：交互式草图条件网格编辑（单次生成，无文本）。

## 6. 主要结论与发现

- **技术评估**：HandMade在20个样例上取得最佳平均几何对齐（F-score@0.05=0.727，高于其他所有基线）。Text-only和Single-view消融显示草图和多视图条件对保留空间结构必不可少。Instant3dit是第二强的基线，但视觉检查显示其可能重新解释非常规布局或弱化部分关系。
- **用户研究主要发现**：
  1. 参与者开始创作时意图通常是部分清晰、部分模糊的；生成结果帮助定义具体形式。
  2. 3D草图适合表达部件布局和空间关系，语言适合指定身份、材质、颜色等语义；两者互补。
  3. 徒手VR草图对粗略结构有效，但精确细节（如平坦表面、小结构）绘制困难。
  4. 中间检查点（多视图图像、最终网格）暴露了不同阶段的失败，允许用户进行局部修复而非重新开始。
  5. 修订主要在文本或重新生成层面进行，无需重新绘制VR草图，体现了草图的锚定作用。
- **总体**：HandMade实现了“空间提示”的交互范式，将早期人类空间意图转化为现有生成模型能使用的条件，降低了空间布局表达的门槛。

## 7. 优点

- **方法创新**：首次将粗糙的、带部分标签的VR 3D草图作为空间提示（而非精确几何）与语言结合，利用现有多视图图像生成和图像到3D重建模型，避免了训练专用模型的数据和算力需求。
- **模块化管线**：各阶段独立（草图渲染、提示构建、图像生成、3D重建），便于替换后端模型；支持迭代修改（草图修订、文本修订、重新生成）。
- **全面评估**：定量比较多种基线，包括消融实验和9种指标；用户研究提供了实际使用中的行为洞察，验证了分工的有效性。
- **可操作中间表示**：多视图部分引导图不仅传给模型，也允许用户在不同阶段识别意图漂移并进行局部修复。
- **设计原则清晰**：支持不精确输入、保留部分级意图、分离空间与语义工作、保持开放域、暴露检查点、生成下游可用资产。

## 8. 不足与局限

- **输入限制**：VR草图仍是有噪声、不完整的，可能受手部跟踪误差、深度控制困难、绘画技能影响；系统无法区分意图模糊与输入误差。
- **管线依赖外部模型**：图像生成阶段可能忽略引导颜色、通过类别先验重新解释非常规比例、产生多视图不一致；3D重建可能平滑薄结构、合并相邻部件、丢失空洞或隐藏几何。
- **技术评估局限性**：仅对比了有公开代码和可适配输入格式的方法，未覆盖更广泛的草图到3D方法（如VRsketch2Gaussian、SketchTo3DGen）；几何对齐指标仅衡量空间一致性，不评估美学、可编辑性、下游可用性。
- **用户研究规模小**：8名参与者，主要为非专家，结果可能不推广到专业创作者；没有进行系统间的受控比较。
- **交互负担**：依赖键盘快捷键、模式切换；5/8参与者提到了控件或视点问题（如键盘回忆、模式切换、头晕、空间感弱）。尚未实现全自动化循环。
- **缺乏空间约束支持**：用户无法指定诸如“接触”、“分离”、“平齐”、“平行”等部分间关系约束，只能通过文本隐式描述。
- **导向参数不可调**：当前无法控制图像生成模型对引导图的忠实程度（仅能通过提示措辞间接调整）。
- **未评估对下游编辑的影响**：生成的网格拓扑未必适合专业编辑，需要重拓扑工具辅助。
