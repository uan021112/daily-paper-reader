## 摘要
基于文本和图像的3D模型如今能够生成令人信服的资产，但它们对于物体应占据或避开的空间提供的直接控制仍然有限。在创作中，这种空间意图通常在生成开始之前就已明确。一把椅子应贴合座椅轮廓，一个道具应留出运动空间，或者一个部件应暴露接触面。提示词和图像视图难以承载此类约束，因此需要显式的控制界面。
我们提出了Arbor，一种可训练的附件，用于文本条件下的潜在3D生成。Arbor将约束网格作为原生3D控制界面引入。该界面使用几何应存在的包络区域、应保持空白的避让区域以及物体应接触的接触区域。与补全或整体物体骨架控制不同，这些网格并非目标证据。它们是局部类型化的要求，并且可以包含不应出现表面的区域。Arbor通过将约束网格转换为标记，并在冻结的去噪器内学习路由附件，从而将此信号保持为几何形式。因此，每个潜在区域都能接收对其空间位置重要的约束部分。
我们在自动和艺术家策划的控制基准上评估了Arbor，涉及包络、避让和接触约束，并将指标趋势与用户偏好研究进行了比较。即使没有专门的合规损失，Arbor也能在保持固定约束下的物体质量和多样性的同时，提高约束服从性。

## Abstract
Text and image conditioned 3D models now generate convincing assets, but they still offer little direct control over the space an object should occupy or avoid. In authoring, this spatial intent is often known before generation starts. A chair should fit a seating envelope, a prop should leave clearance for motion, or a part should expose a contact surface. Prompts and image views are poor carriers for such constraints, requiring the need for an explicit control interface.   We present Arbor, a trainable attachment for text conditioned latent 3D generation. Arbor introduces constraint meshes as a native 3D control interface. The interface uses hull regions where geometry should exist, avoidance regions that should remain empty, and touch regions the object should contact. Unlike completion or whole object scaffold control, these meshes are not target evidence. They are local typed requirements and can include regions where no surface should appear. Arbor keeps this signal as geometry by converting constraint meshes into tokens and learning a routed attachment inside a frozen denoiser. Each latent region can therefore receive the part of the constraint that matters for its spatial location.   We evaluate Arbor on automatic and artist curated control benchmarks with hull, avoidance, and touch constraints, and compare the metric trends to a user preference study. Even without dedicated compliance losses, Arbor improves constraint obedience while preserving object quality and variation under fixed constraints.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有文本/图像条件3D生成模型（如TRELLIS、Hunyuan3D等）能够生成高质量资产，但缺乏对物体**空间占用**的直接控制。用户无法指定物体应占据（hull）、避开（avoidance）或接触（touch）的特定3D区域。文本提示描述语义而非空间，图像条件受限于视角，均难以承载精确的几何约束。
- **背景与动机**：在游戏、影视等工业场景中，3D资产常需配合预定空间（如椅子需适配坐姿轮廓、道具需留出动画运动空间）。当前工作流依赖艺术家反复生成并手动修改，效率低下。已有工作（如SIGNeRF、SpaceControl、SK-Adapter等）分别解决编辑、采样时引导或骨架控制，但缺乏原生、类型化的几何约束接口。
- **整体含义**：Arbor提出以**约束网格**作为原生3D控制界面，将hull、avoidance、touch三类区域编码为紧凑token，通过路由附件注入冻结的扩散骨干，使生成同时遵循文本提示和空间约束，显著提升可控性和生产力。

## 2. 论文提出的方法论：核心思想、关键技术细节

**核心思想**：将艺术家制作的约束网格（hull/avoidance/touch）转换为几何条件token，在冻结的TRELLIS文本条件骨干中学习一个轻量路由附件，使每个隐式格子只关注其空间位置相关的约束证据。

**关键技术细节**：

- **骨干模型**：采用TRELLIS-1 (T1) 作为文本条件稀疏结构生成器（12层Transformer，16³格子，768宽），其权重冻结；使用TRELLIS-2 (T2) 的冻结编码器将约束网格编码为紧凑token（32³格子，32维）。
- **约束编码**：
  - 将hull/avoidance/touch三类网格融合为单一网格 `C_mesh`，附有表面法线和类型信号（3个二值通道）。
  - 通过T2的 OVoxelize 得到形状场和属性场，再用T2的shape encoder和material encoder分别编码约束形状和信号（法线+类型），得到几何记忆 `c_geo`。
  - 将 `c_geo` 投影到T1模型宽度，并加入学习的位置嵌入。
- **路由机制（Router）**：
  - 将T1的16³格点按4×4×4划分为64个组。
  - 每组通过其8个角点作为锚点，从全部几何token中选出最近的K=2048个作为本地上下文 `c_local`。
  - 另外用96个学习摘要token通过交叉注意力聚合全局信息得到 `c_global`。
  - 每组的上下文为 `c_ctx = [c_local; c_global]`。
- **适配器（Adapter）**：
  - 在每个T1块内，在文本交叉注意力和FFN之间插入几何残差分支：
    \[ \Delta z_{geo}^{(l,g)} = \text{CrossAttn}_{geo} \left(z_{\text{text}}^{(l,g)}, c_{ctx}^{(g)}\right) \]
    \[ z^{(l,g)} = \text{FFN}^{(l)} \left(z_{\text{text}}^{(l,g)} + W_{geo}^{(l)} \Delta z_{geo}^{(l,g)} \right) \]
  - `W_geo` 零初始化，保证初始不影响预训练输出。
- **训练**：仅训练几何投影、位置嵌入、全局摘要、路由适配器等面向几何的模块，其余冻结。优化采用T1原始流匹配损失，无额外约束损失。文本和几何条件独立随机丢弃以实现无分类器引导。

- Arbor显著提升约束服从性（Hull Hit、Touch Hit等），同时保持生成质量和多样性（Ctrl. Score、MV-CLIP、用户偏好均优于所有基线）。

- 约束扫描表明控制平滑连续，不会塌缩为少量布局模式。

## 7. 优点

- **方法创新性**：首次将hull/avoidance/touch三类类型化约束网格作为原生3D控制接口，覆盖“必须占据”“必须避开”“必须接触”三种常见空间需求。
- **工业友好性**：适配器设计使冻结预训练骨干不受影响，仅添加轻量模块，训练开销小（80小时8卡），易于集成到现有管线。
- **路由机制巧妙**：通过局部TopK+全局摘要解决密集几何token与潜在格点的分辨率不匹配问题，保证计算量可控且空间相关。
- **自动约束生成系统**：在线随机组合多种约束（Part Union、随机面片、布局阻挡等），弥补了无标注约束数据的空白，提升训练多样性和稳定性。
- **实验设计全面**：同时包含自动/人工基准、用户研究、消融、扫描、变化性对比，内部变体比较，评估维度完整。
- **开源友好**：计划发布基准清单、评估代码等，促进后续研究。

## 8. 不足与局限

- **仅控制稀疏结构阶段**：Arbor作用于T1的稀疏结构（16³格点），后续SLAT精炼和网格解码阶段未受直接控制，局部细节可能被软化。
- **约束与文本冲突**：当约束网格与文本提示语义严重冲突（如极小hull、激进避让），模型可能在局部服从与全局语义间产生妥协。
- **人工基准规模小**：手动分割仅32例，且仅限于Toys4K类别，泛化性待验证。
- **用户研究样本有限**：27人、404试，且仅统计胜率，未提供显著性检验，结论强度有限。
- **未与大规模训练充分对齐**：训练数据量仅为TRELLIS的10%，若用更多数据可能进一步改善。
- **语义标签尚未成熟**：Arbor Semantics（给几何区域附加部分标签）未超越纯几何路由，文本路径注入语义不如预期有效，该方向仍需探索。
- **对比公平性争议**：变化性实验中，Point-E等接受图像输入，任务不同；采样时引导基线（SpaceControl、Gradient）在生成质量上明显弱于Arbor，可能为预训练直接提供编码优势。
- **编码依赖TRELLIS-2**：需预先安装TRELLIS-2的冻结编码器，架构耦合度较高。
