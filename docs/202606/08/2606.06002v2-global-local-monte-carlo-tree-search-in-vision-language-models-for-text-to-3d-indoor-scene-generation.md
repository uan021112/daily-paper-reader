## 摘要
大型视觉-语言模型在各种任务中取得了显著的推理性能。然而，关于使用LVLMs进行文本到3D室内场景生成的研究仍较少。主要挑战在于，现有的基于LVLM的方法采用链式思维顺序决策机制，无法修正早期决策，导致错误传播。本文将任务视为一个受空间和布局常识约束的规划问题。为解决该问题，我们将其建模为一个具有全局树和局部树的树搜索问题，这不同于现有的顺序决策方法。在全局树中，我们迭代地放置每个物体，并像人类布置房间一样探索多次尝试，其中问题空间被表示为树。为了有效搜索该树，我们提出了一种层次化场景表示和PRM引导的MCTS方法。这种表示将场景抽象为房间级别、区域级别、地板物体级别和支撑物体级别。PRM引导的MCTS方法使用PRM来修剪不必要的分支，并使用MCTS算法平衡探索与利用，从而以更少的尝试获得最优解。在局部树中，它进一步将每个物体的放置分解为更细的子步骤，包括具体的放置参数。为了使整个场景外观一致，我们利用预训练的扩散图像生成模型为场景中的所有物体预测纹理。由于现有的文本到3D室内场景生成基准在规模和多样性上仍然有限，我们收集了一个新的大规模多样化数据集，包含65种场景类型和3250条指令，具有不同的尺寸、布局和风格，命名为3DTindo-bench，以更好地评估最先进模型的能力。我们的实验表明，我们的方法比最先进的方法生成更逼真的3D场景。

## Abstract
Large Vision-Language Models have achieved significant reasoning performance in various tasks. However, there are few studies on text-to-3D indoor scene generation with LVLMs. The main challenge is that prevailing LVLM-based methods employ chain-of-thought sequential decision mechanisms that cannot revise earlier decisions, causing error propagation. In this paper, we consider the task as a planning problem constrained by spatial and layout commonsense. To solve this problem, we model it as a tree search problem with global and local trees, which differs from existing sequential decision-making approaches. In the global tree, we place each object iteratively and explore multiple attempts like humans furnishing a room, where the problem space is represented as a tree. To effectively search the tree, we propose a hierarchical scene representation and a PRM-guided MCTS method. This representation abstracts a scene into room level, region level, floor object level, and supported object level. The PRM-guided MCTS method uses the PRM to prune unnecessary branches and the MCTS algorithm to balance exploration and exploitation to get an optimal solution with fewer attempts. In the local tree, it further decomposes the placement of each object into finer sub-steps, including the specific placement parameters. To make the whole appearance of the scene consistent, we leverage pre-trained diffusion image generative models to predict textures for all the objects in the scene. As existing benchmarks for text-to-3D indoor scene generation remain limited in scale and diversity, we collect a new large-scale diverse dataset that contains 65 scene types and 3250 instructions with diverse sizes, layouts, and styles, named 3DTindo-bench, to better assess the capability of the state-of-the-art models. Our experiments show that our method generates more realistic 3D scenes than state-of-the-art methods.

## 1. 核心问题与整体含义

- **研究动机**：现有基于大型视觉-语言模型（LVLM）的文本到3D室内场景生成方法大多采用链式思维顺序决策机制（如LayoutGPT、HoloDeck等）。这种机制在推理时从左到右依次放置物体，一旦早期决策（如某个物体位置）错误，后续生成会累积误差且无法修正，导致布局不合理。
- **核心问题**：如何让LVLM在布局生成中具备类似人类“试错—回溯”的能力，即不是线性一步到位，而是在一个解空间（树结构）中搜索最优布局。
- **整体含义**：本文将该任务视为一个受空间和布局常识约束的规划问题，建模为全局树和局部树的搜索问题，并提出PRM（过程奖励模型）引导的蒙特卡洛树搜索（MCTS）框架，以高效探索解空间、避免错误传播，从而生成更真实、布局更合理的3D室内场景。

## 2. 方法论

### 核心思想
- 将布局生成过程从链式顺序改为树结构搜索。每个物体放置时存在多种候选位置/姿态，整体构成一棵树。通过搜索从根（空布局）到叶（完整布局）的最优路径，并允许回溯修正。
- 引入层次化场景表示作为中间代理，将复杂场景分解为房间、区域、地板物体、支撑物体四个层级，降低搜索复杂度。
- 使用PRM评估中间状态（部分布局）并剪枝，结合MCTS平衡探索与利用，最终在有限迭代内找到高质量布局。
- 利用预训练扩散图像生成模型（如Stable Diffusion + ControlNet）对所有物体重纹理，保证场景外观一致性。

### 关键技术细节
- **层次化场景表示**：
  - **房间级**：确定房间尺寸（长、宽）。
  - **区域级**：将房间分为多个功能区域（如休息区、存储区），每个区域有独立尺寸和布局。
  - **地板物体级**：每个区域包含放置在地板上的物体（如床、沙发），以及它们与区域锚点物体（如床是休息区的锚点）的空间关系。
  - **支撑物体级**：放置在地板物体上的小型物体（如闹钟在床头柜上）。
- **全局树与局部树**：
  - **全局树**：节点表示已放置物体的集合（布局状态），边表示放置一个新物体。每个物体有多个候选放置方案，构成树的分支。
  - **局部树**：对于每个新物体的放置，进一步分解为三个子步骤：确定相对于锚点的 **侧边**（左/右/顶/底）、**行索引**、**列索引**。使用emoji网格可视化当前布局，辅助LVLM进行空间推理。
- **PRM引导的MCTS**：
  - 修改标准MCTS的选择步骤：使用PRM-UCB公式代替UCT，其中PRM得分低于阈值δ的节点被赋值为 \(-\infty\)，即直接剪枝。
  - 公式：
    \[
    \text{PRM-UCB}(C', C; \delta) = 
    \begin{cases}
    \overline{v}_{C'} + \epsilon \sqrt{\frac{\ln n_C}{n_{C'}}}, & \text{if } r_{\text{prm}}^{C'} \ge \delta \\
    -\infty, & \text{if } r_{\text{prm}}^{C'} < \delta
    \end{cases}
    \]
  - 其中 \(r_{\text{prm}}^{C'}\) 由PRM（使用LVLM评估中间布局质量）给出。
  - 扩展步骤：生成k个子节点（k=2）。
  - 模拟步骤：执行n次随机rollout（n=5），用ORM（结果奖励模型）评估终端状态，取平均值。
  - 回传步骤：将模拟值沿路径反向传播更新节点统计量。
- **纹理生成**：
  - 先检索：从Objaverse数据库中根据类别、尺寸、CLIP相似度等检索3D资源。
  - 再重纹理：使用深度条件扩散模型分别生成物体正面和背面纹理，并通过混合系数λ（论文设为0.3）平衡视图一致性与可控性，最后在UV空间用修补模型完善。

## 7. 优点

- **方法论创新**：首次将布局生成建模为树搜索，引入PRM引导MCTS，克服链式推理不可回溯的根本缺陷。
- **层次化代理**：通过四层场景表示，将LVLM的常识知识结构化，大幅缩小搜索空间，使MCTS可行。
- **可扩展性**：方法为训练无关，可直接利用任意LVLM，便于迁移到更强模型。
- **基准贡献**：3DTindo-Bench规模大（65类3250指令）、多样性高（尺寸、关系、风格），填补了该领域评估数据空白，推动社区发展。

## 8. 不足与局限

- **依赖外部3D数据库**：若指令指定稀有物体（如“decanters”），Objaverse中缺失，则无法检索，导致物体缺失（如失败案例所示）。放宽至使用生成式3D模型可缓解。
- **美学质量有限**：生成的纹理虽然一致，但美学风格不够精细，LVLM对高级设计原则的理解仍有提升空间。
- **仅使用单LVLM**：实验全部基于Qwen2.5-VL-72B，未验证在其他LVLM（如GPT-4V、LLaVA-NeXT）上的泛化性能，可能存在模型偏差。
- **基准覆盖仍不完美**：3DTindo-Bench虽大，但指令由DeepSeek-V3生成，可能带有生成模型偏好；场景类型分类参考了Neufert和Quattoni等，但未包含所有可能场景（如教堂、飞机内部等）。
- **搜索效率**：虽然PRM减少分支，但MCTS迭代次数（10次）和rollout次数（5次）仍然有限，对于极大场景（>20个物体）可能耗时增加（平均1.87分钟/场景）。
- **缺乏端到端对比**：未与较新的数据驱动方法（如DiffuScene、InstructScene）在公平条件下对比（因为它们需要训练数据），仅与LVLM-based方法对比，略显局促。
