## 摘要
最近的3D重建和编辑系统大多操作在隐式和显式表示上，如NeRF、点云或网格。虽然这些表示能够实现高保真渲染，但它们本质上是低层次的，难以通过编程控制。相比之下，我们提出并系统评估了一种新的3D重建范式——3D代码合成（3D-CoS），其中3D资产被构建为可执行的Blender代码，这是一种可编程且可解释的媒介。为了评估当前VLM在代码表示3D对象方面的能力，我们在统一协议下评估了具有代表性的开源和闭源VLM在基于代码的重建中的表现。我们进一步引入了一套结构化代码合成工作流程，包括基于蓝图的规划、针对Blender API文档的检索增强生成（RAG）、少样本几何演示以及用于逐部分代码生成的组件级Agent工作流。为了展示这种表示的独特优势，我们进一步评估了局部文本驱动的修改，并将我们的基于代码的编辑与基于点云的3D编辑基线进行了比较。我们的研究表明，代码作为3D表示具有强大的可控性和局部性，在目标编辑评估中产生了更强的编辑保真度和更好的未编辑区域保留。我们的工作还分析了这一范式的潜力，描绘了当前VLM在程序化3D建模方面的能力边界，并强调了代码合成作为可编辑3D重建的一个有前景的方向。

## Abstract
Most recent 3D reconstruction and editing systems operate on implicit and explicit representations such as NeRF, point clouds, or meshes. While these representations enable high-fidelity rendering, they are fundamentally low-level and hard to control programmatically. In contrast, we propose and systematically evaluate a new 3D reconstruction paradigm, 3D Code Synthesis (3D-CoS), where 3D assets are constructed as executable Blender code, a programmatic and interpretable medium. To assess how well current VLMs can use code to represent 3D objects, we evaluate representative open-source and closed-source VLMs in code-based reconstruction under a unified protocol. We further introduce a suite of structured code-synthesis workflows, including blueprint-based planning, Retrieval-Augmented Generation (RAG) over Blender API documentation, few-shot geometric demonstrations, and a component-level Agent workflow for part-wise code generation. To demonstrate the unique advantages of this representation, we further evaluate localized text-driven modifications and compare our code-based edits with a point-cloud-based 3D editing baseline. Our study shows that code as a 3D representation offers strong controllability and locality, yielding stronger edit fidelity and better preservation of unedited regions in our targeted editing evaluation. Our work also analyzes the potential of this paradigm, delineates the current capability frontier of VLMs for programmatic 3D modeling, and highlights code synthesis as a promising direction for editable 3D reconstruction.

*   **整体含义**：论文提出并系统评估了一个全新的3D重建范式 **3D-CoS (3D Code Synthesis)**。该范式将3D资产构建为可执行的**Blender Python脚本**，将重建问题转化为代码合成问题。通过让VLM生成这种程序化的、可解释的代码，论文试图将3D重建从“恢复几何”提升到“恢复设计程序”的高度，从而在可控性、可编辑性和可解释性上获得根本性优势。

### 二、论文提出的方法论

*   **核心思想**：将3D重建视为“从图像到代码”的映射。输入一张单视角图像，VLM生成可执行的Blender Python脚本，脚本渲染出的3D物体应尽可能匹配目标。代码作为中介表示，实现了几何结构、布局和外观的参数化、显式建模。
*   **关键技术细节**：论文探索了多种代码合成工作流，旨在提升VLMs生成代码的质量和鲁棒性。
    1.  **单次调用 (Single-call)**：最直接的方式，VLM一次性生成完整的Blender脚本。速度最快，但容易遗漏细节或参数错误。
    2.  **蓝图规划 (Planning)**：将任务分解为“规划”和“执行”两阶段。
        *   **阶段1 (蓝图生成)**：VLM从输入图像推断出一个量化的蓝图（Blueprint），定义物体各部件的相对尺寸和位置关系，用结构化JSON格式表示。
        *   **阶段2 (代码生成)**：VLM在蓝图的约束下生成最终的Blender代码。这实现了“要做什么”（几何规划）与“如何做”（API调用）的分离。
    3.  **检索增强生成 (RAG)**：为解决VLM可能幻觉API调用或使用不当函数的问题，该方法在代码生成时引入外部知识。
        *   首先爬取并索引完整的Blender 4.4 API文档（覆盖21,102个函数）。
        *   根据蓝图中的每个组件，VLM生成API查询，从Qdrant向量数据库中检索最相关的文档片段作为上下文，以此指导代码生成。
    4.  **少样本演示 (Few-shot)**：在任务指令前，预先提供少量（由专家模型生成并精选的）图像-代码对作为示例。引导VLM通过上下文学习，隐式掌握几何推理模式和API使用规范。
    5.  **组件级Agent工作流 (Agent)**：引入一个更复杂的代理系统。
        *   **分解**：VLM先将输入图像分解成一个由多个部件（Part）组成的结构计划。
        *   **逐部件生成**：为每个部件单独合成代码。
        *   **执行与纠错**：在生成过程中执行代码，若执行失败，Agent会进行修正并重新执行，直到成功。
        *   **组装**：将所有成功的部件程序组装成完整的3D模型。此方法特别有利于提高局部几何保真度和复杂组件的平衡恢复。
*   **算法流程（文字说明）**：
    1.  **输入**：单张RGB图像 **I**。
    2.  **处理**：将 **I** 输入到VLM中，结合四种不同的提示范式或Agent工作流之一。
    3.  **代码生成**：VLM推理并生成可执行的Blender Python脚本。
    4.  **编译与渲染**：执行代码，编译生成3D网格或点云，并进行渲染。
    5.  **评估**：通过计算3D和2D指标来评估重建质量，并基于生成的代码进行文本驱动的编辑测试。

### 三、实验设计

*   **数据集与基准**：
    1.  **重建基准**：基于 **ModelNet10** 数据集，从中采样100个物体，每个物体从8个视角渲染，由人工选择最具信息量的单视图作为输入。数据集还切分为“Easy”和“Hard”子集，以分析结构复杂度的影响。
    2.  **编辑基准**：基于 **BlendNet** 数据集构建了 **BlendNet-E**，包含55个对象及其手动编写的高质量文本编辑指令。
    3.  **变体基准**：**ModelNet10-V**，为每个ModelNet10图像生成一个编辑指令，用于定性地文本条件重建。
*   **对比方法**：
    1.  **传统网格基线**：**Unique3D**、**InstantMesh**（代表当前先进水平的专用重建模型）。
    2.  **基于代码的基线**：**MeshCoder**（需要真实点云输入）。
    3.  **VLM基线**：包括3个开源族（InternVL3.5 38B, LLaVA-OneVision 72B, Qwen2.5-VL 72B）和3个闭源族（Claude Sonnet 4, o3, Gemini 3 Pro）。所有VLM都使用4种直接提示范式，闭源VLM额外使用了Agent工作流。
*   **评估指标**：
    1.  **3D指标**（计算前先进行姿态对齐）：Chamfer距离（CD）、5%阈值下的F-score (F@5%)、2%阈值下的空间平衡召回率 (SBR@2%)、5%阈值下的法线一致性 (NC@5%)。
    2.  **2D指标**（在输入视角下计算）：轮廓IoU (Sil IoU)、深度归一化RMSE (NRMSE)、法线平均角度误差 (MAE)。
    3.  **编辑指标**：CLIP相似度 (CLIP sim) 和CLIP方向性 (CLIP dir)，以及用户偏好研究。

*   **充分性与公平性**：
    *   **充分性**：实验覆盖了从开源到闭源、从简单直接提示到复杂Agent工作流的多种场景，评估维度全面（3D几何+2D视图+编辑能力），结论扎实可靠。

### 六、论文的主要结论与发现

1.  **代码重建具有竞争力**：由顶尖闭源VLM（Gemini 3 Pro）驱动的代码合成在重建质量上**超越了Unique3D和MeshCoder**，并**接近InstantMesh**。例如，Gemini 3 Pro 在Few-shot范式下的CD从MeshCoder的0.0442降至0.0249。
2.  **VLM能力是当前瓶颈**：在相同范式下，闭源VLM的性能显著优于开源VLM，表明当前代码重建的上限主要受限于VLM的**感知、推理和代码生成能力**，而非代码表示的潜力。
3.  **结构化工作流效果显著**：不同VLM对不同辅助结构（Planning, RAG, Few-shot, Agent）反应不同。没有一种范式是通用的最优解。例如，Gemini 3 Pro在所有结构化范式中表现都强，o3在Planning和Few-shot下表现最佳，Qwen2.5-VL在Few-shot下提升最大。
4.  **Agent擅长提升局部覆盖**：Agent工作流在空间平衡召回率（SBR@2%）指标上表现突出，表明它能更均衡地恢复物体的各个部件，但可能在全局一致性上做出牺牲。
5.  **代码表示在编辑上的核心优势**：在局部文本驱动编辑任务中，基于代码的编辑方法（使用o3）**显著优于**点云编辑基线（BlendedPC），在CLIP相似度上高出4.1倍，在CLIP方向性上高出23.9%，且用户研究也印证了其在指令遵循和未编辑区域保留上的优势。

### 七、优点

*   **创新性**：开创性地将3D重建问题重新定义为VLM代码合成问题，提出了一种全新的、可解释、可编辑的3D表示范式，与传统方法形成鲜明对比。
*   **系统性**：构建了包含多种提示范式、Agent工作流、RAG和少样本学习的完整方法论体系，对VLM在程序化3D建模方面的能力进行了全面而系统的评估。
*   **评估的严谨性**：设计了全面的评估协议，包括姿态对齐、多指标（3D+2D）、置信区间、Easy/Hard分割、失败率报告等，确保了实验结果的客观性和公平性。
*   **编辑优势的实证**：通过定量指标和用户研究强力证明了代码表示在3D编辑任务中的独特优势，为后续研究提供了有力支撑。

### 八、不足与局限

*   **与专用模型的差距**：尽管接近，但最好的代码合成方法（Gemini Agent/Near-shot）在聚合指标上仍未完全超越诸如InstantMesh的专用3D重建模型，表明代码合成在恢复连续、精细曲面方面仍有差距。
*   **Agent工作流的局限**：Agent在逐部件生成时，可能出现部件间缩放、放置或相互一致性问题，导致**全局协调错误**，并非在所有情况下都优于直接提示。
*   **实验范围局限**：重建实验主要集中在**ModelNet10**数据集上，该数据集以几何形状为主，纹理简单。在更复杂、纹理丰富的真实世界图像上的表现有待验证。
*   **编辑评估范围较小**：与基线的编辑比较仅在“灯”和“桌子”两个类别上进行，泛化能力结论的稳健性有待在更多类别上确认。
*   **资源消耗与效率**：论文未讨论VLM代码合成的推理速度、计算成本（Token消耗）以及执行Blender代码的渲染时间。Agent等工作流可能非常慢，不适用于实时或需要快速迭代的场景。
*   **依赖VLM的能力**：整个范式的性能上限依赖于VLM的感知和推理能力，VLM的幻觉、不准确的空间理解或API误用都会直接导致重建失败。自我验证和纠错机制仍需加强。
