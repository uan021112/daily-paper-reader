## 摘要
我们研究了从局部点云序列进行4D重建的问题，其中深度传感器观测数据不完整、无序且缺乏显式的时间对应关系。这种纯几何设置因缺失观测和模糊动态而具有挑战性。虽然近期进展主要依赖于基于图像的方法，但现有的基于点云的方法通常聚焦于单一物体、假设相对完整的输入或需要显式对应关系。为解决这些限制，我们提出DynaTok，一种基于点云的框架，用于从局部点云序列进行无需对应关系的4D重建，且无需图像。DynaTok将帧编码为紧凑的潜在token，通过基于Transformer的时空编码器随时间聚合不完整的观测，并通过残差token在统一模型中解耦几何与运动。然后，一个流匹配解码器根据潜在token重建完整、时序一致的4D点云序列。在物体级和场景级基准上的实验表明，从局部点云观测中重建质量和时序一致性有所提升。项目页面：https://wrchen530.github.io/dynatok/。

## Abstract
We address 4D reconstruction from partial point cloud sequences, where depth-sensor observations are incomplete, unordered, and lack explicit temporal correspondences. This geometry-only setting is challenging due to missing observations and ambiguous dynamics. While recent progress has largely relied on image-based methods, existing point-based approaches typically focus on single objects, assume relatively complete inputs, or require explicit correspondences. To address these limitations, we propose DynaTok, a point-based framework for correspondence-free 4D reconstruction from partial point cloud sequences without images. DynaTok encodes frames into compact latent tokens, aggregates incomplete observations over time with a Transformer-based spatiotemporal encoder, and decouples geometry and motion through residual tokens in a unified model. A flow-matching decoder then reconstructs complete, temporally consistent 4D point-cloud sequences conditioned on the latent tokens. Experiments on object- and scene-level benchmarks demonstrate improved reconstruction quality and temporal coherence from partial point cloud observations. Project page: https://wrchen530.github.io/dynatok/.

# 1. 核心问题与整体含义

- **研究动机**：在机器人、增强现实等应用中，需要从深度传感器等几何观测中重建动态场景。但实际输入往往是局部点云序列——不完整、无序、且帧间无点对点对应关系。现有4D重建方法大多依赖图像或视频中的丰富外观线索，且常要求完整输入或显式对应；而基于点云的方法通常局限于静态场景或单物体，且需要完整点云或规范模板。
- **核心问题**：如何仅从部分、无序、无对应关系的点云序列中恢复完整、时序一致的4D动态场景（包括静态背景和运动物体）？关键在于时序聚合：如何利用多帧互补信息填补缺失区域，并隐式区分静态结构与动态运动。
- **整体含义**：DynaTok是首个无需图像、无需显式对应关系、仅靠点云监督的通用4D重建框架，同时适用于物体级和场景级动态场景，填补了纯几何4D重建的空白。

# 2. 方法论

- **核心思想**：将每帧部分点云编码为一组紧凑的潜tokens，通过Transformer进行跨帧时空对齐，再利用残差机制将聚合后的表示分解为几何token（参考帧，捕获持久结构）和运动token（后续帧，捕获时序变化），最后用条件流匹配解码器从潜tokens重建完整点云。
- **关键技术细节**：
  - **逐帧token提取**：对每帧点云使用最远点采样（FPS）选取M个锚点查询，通过交叉注意力（点坐标经傅里叶嵌入后作为Key/Value）和自注意力获得M个D维token。
  - **时空对齐模块**：由多个交替的帧内自注意力和跨帧全局注意力层组成，使token能在帧间传播信息，隐式建立跨帧一致性。使用3D旋转位置编码（RoPE）编码空间关系。
  - **残差分解**：以第一帧的token为参考G，后续帧token表示为Z_s = G + H_s，其中H_s为残差。H_1=0。这样将时序变化显式编码为残差，无需额外网络。
  - **条件流匹配解码器**：定义线性插值路径 x_t = (1-t)ε + t x_1（ε为均匀立方体噪声，x_1为目标点云）。解码器估计条件速度场 Φ_dec(x_t, t, Z_s)，训练目标为最小化预测速度与目标速度x_1-ε的L2损失。推理时从噪声积分至t=1获得重建点云。
- **算法流程**：输入部分点云序列X_s → 逐帧提取tokens F_s → 时空对齐模块输出G和H_s（Z_s=G+H_s） → 条件流匹配解码器以Z_s为条件，从噪声生成完整点云Y_s。所有组件端到端训练，仅需点云监督。

# 6. 主要结论与发现

- DynaTok在物体级和场景级4D重建中均显著优于现有方法，尤其在高完整性和时序一致性方面（在DT4D上Completeness<0.03，NC>0.9；Kubric上FG+BG场景NC=0.888）。
- 时空聚合是成功的关键：没有全局注意力的逐帧编码导致性能大幅下降（Acc从0.027→0.276）。
- 残差分解有效地实现了几何与运动解耦，使持久结构共享、变形灵活编码。
- 模型对稀疏输入鲁棒（即使512点也能恢复整体结构），且能零样本泛化到真实场景（基于CUT3R点云）。
- 模型轻量高效（19M参数，每场景22秒），适合实际应用。

# 7. 优点

- **新颖的问题设定**：首次系统研究从局部、无序、无对应关系的点云序列进行4D重建，更贴合实际传感器数据。
- **统一框架**：无需图像、无需水密网格、无需显式对应、无需规范模板，完全基于点云监督，适用性强。
- **有效的token设计**：通过FPS+交叉注意提取紧凑token，降低了序列的变异性；残差分解在单一潜空间中分离几何与运动，简洁高效。
- **条件流匹配解码器**：自然处理可变点数，避免点对点对应，适合无序输出。
- **充分的实验验证**：多个数据集、消融、零样本泛化、模型效率对比，结论可靠。
- **开源承诺**：论文声明将开源代码，利于复现和后续研究。

# 8. 不足与局限

- **实验覆盖局限**：对比方法有限（尤其是4D基线），因为极少方法适配相同设置。与Motion2VecSets比较时，后者仅支持512点输入，而DynaTok使用8192点，输入规模不对等可能影响公平性（论文承认但篇幅所限未调整）。
- **对长时间遮挡和极快运动敏感**：论文在A.5节讨论失败案例，指出长时间遮挡、快速运动、重物体重叠、高度非刚性变形可能导致重建退化。
- **训练数据限制**：Kubric主要包含刚性/近刚性物体，模型对高度可变形物体（如布料、液体）可能泛化不足。
- **零样本实验仅定性**：真实场景泛化仅提供定性结果，缺乏定量指标（如GT点云不可得），说服力有限。
- **推理速度**：虽比TRELLIS快，但流匹配需要25步ODE积分，难以达到实时（22秒/16帧场景）。
- **输入规模假设**：训练使用每帧8192点，实际传感器可能更稀疏或更密集，模型在不同输入分辨率下的泛化能力未充分测试。
- **缺少多模态融合深度分析**：RGB+DINOv2实验仅作为附录，未在主实验中进行系统比较，且DINOv2不针对3D几何优化。
