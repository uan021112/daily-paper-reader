## 摘要
开放词汇的3D功能分割使机器人能够在3D场景中定位功能对象组件。这是一项需要空间理解和任务解释的挑战性任务。当前的开放词汇3D分割方法主要关注对象级识别，而场景级部件分割方法试图穷尽分割整个场景，导致资源密集且耗时。在粒度、准确性和速度之间平衡分割性能仍然是一个挑战。为缓解这一问题，我们引入了T-FunS3D，一种任务驱动的分层开放词汇3D功能分割方法，为机器人应用提供可操作的感知。我们的方法以室内场景的3D点云和带有姿态的RGB-D图像作为输入。通过提取环境中的实例及其视觉嵌入，构建开放词汇场景图。给定任务描述，T-FunS3D识别场景图中最相关的实例，并利用视觉-语言模型定位其功能组件。在SceneFun3D数据集上的实验表明，T-FunS3D在开放词汇3D功能分割方面与最先进方法相当，同时实现了更快的运行时间和更低的内存使用。

## Abstract
Open-vocabulary 3D functionality segmentation enables robots to localize functional object components in 3D scenes. It is a challenging task that requires spatial understanding and task interpretation. Current open-vocabulary 3D segmentation methods primarily focus on object-level recognition, while scene-wide part segmentation methods attempt to segment the entire scene exhaustively, making them highly resource-intensive and time consuming. Balancing segmentation performance in terms of granularity, accuracy, and speed remains a challenge. As one step towards alleviating this, we introduce T-FunS3D, a task-driven hierarchical open-vocabulary 3D functionality segmentation method that provides actionable perception for robotic applications. Our method takes as input the 3D point cloud and posed RGB-D images of an indoor scene. We construct an open-vocabulary scene graph by extracting instances and their visual embeddings in the environment. Given a task description, T-FunS3D identifies the most relevant instances in the scene graph and locates their functional components leveraging a vision-language model. Experiments on the SceneFun3D dataset demonstrate that T-FunS3D is comparable to state-of-the-art in open-vocabulary 3D functionality segmentation, while achieving faster runtime and reduced memory usage.

- **问题**：开放词汇的3D功能分割要求机器人根据自由形式任务描述，在3D场景中定位交互功能部件（如门把手、旋钮）。现有方法存在明显不足：

- **动机**：提出一种**任务驱动、分层**的高效方法，仅对与任务相关的对象进行部件分割，避免场景级冗余处理，同时保持开放词汇的泛化能力。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **两阶段处理**：
  - **第一阶段（一次构建）**：从输入的3D点云和带姿态的RGB-D图像中，提取**类无关实例**，并为每个实例计算**开放词汇视觉嵌入**，构建**开放词汇场景图**（节点含视觉嵌入，边在查询时动态计算）。
  - **第二阶段（按需查询）**：给定任务描述，利用**大语言模型（LLM）**解析出上下文对象、功能部件及空间关系，在场景图中定位上下文对象，然后结合**视觉语言模型（VLM）**对其功能部件进行2D分割并提升至3D。

### 关键技术细节
1. **开放词汇实例分割**：
   - 使用**Mask3D**生成类无关掩码，仅保留高置信度提议。
   - 对每个实例选取K个最优视角，在每个视角上生成**全尺寸图像**、**对象裁剪**和**掩码裁剪**（去除背景干扰）。
   - 使用**FG-CLIP**视觉编码器提取上述三种图像的嵌入，并取平均作为该实例的最终视觉嵌入。
2. **场景图构建**：
   - 节点：包含实例的**视觉嵌入**。
   - 边：**查询时动态计算**——将候选上下文对象节点和指代对象节点对应的全尺寸图像嵌入取平均，作为边嵌入，用于匹配任务描述中的空间关系。
3. **任务理解（LLM）**：
   - 使用**Qwen3-14B**（禁用思考模式）解析任务描述，输出JSON格式：上下文对象C、功能部件F、空间关系S、指代对象R。
4. **上下文对象定位**：
   - 用FG-CLIP文本编码器对C和R编码，与场景图节点嵌入计算**余弦相似度**，选出候选节点。
   - 若有空间关系，则进一步在候选节点对之间计算边嵌入与空间关系文本嵌入的相似度，确定最终节点。
5. **功能部件分割**：
   - 对定位到的上下文对象，将其对应视角图像输入**Molmo-7B-D**，以任务描述为提示，输出功能部件像素坐标。
   - 将该坐标输入**SAM**生成2D掩码，**选取最小的掩码**（而非最高置信度），避免分割整个对象。
   - 将2D掩码反投影到3D点云，通过多视角投票聚合得到最终3D功能分割。

## 7. 优点

- **完全training-free**：无需为每个场景或特定任务训练模型，直接利用预训练VLM和LLM。
- **任务驱动的高效分层**：只分割任务相关的对象部件，避免场景级冗余计算，适合实际机器人部署。
- **场景图支持复杂推理**：通过编码视觉嵌入到节点和边，实现了开放词汇的指代定位，无需显式文本描述的关节度计算。
- **视觉嵌入增强**：通过多视角、多尺度（原图、裁剪、掩码裁剪）融合，提高了鲁棒性，减少背景干扰。
- **细粒度控制**：设计性地选择最小SAM掩码，避免部件被整个对象掩码覆盖，提升了功能分割的精确度。

## 8. 不足与局限

- **误差累积**：多阶段流水线（实例分割→场景图→LLM解析→VLM定位→SAM分割）中前序模块的错误会传播到后续，例如小物体（开关、插座）在实例阶段就未被分割。
- **依赖视角质量**：最终3D分割依赖于观察视角的覆盖率和质量，大斜角视图会导致精度下降；当前缺乏主动探索机制。
- **远程关系局限**：方法假设功能部件与对象存在物理连接，无法处理“天花板灯-墙开关”这类远距离、非视觉直接连接的关系。
- **增量更新未解决**：场景图构建假设环境静态，未探讨如何高效新增或更新节点/边以适应环境变化。
- **计算资源门槛**：尽管比Fun3DU高效，但阶段①仍需约10分钟（580s）处理一个场景，阶段②仍需78s/查询，依赖GPU（A40）和大模型（Qwen3-14B、Molmo-7B），对低端机器人平台仍显沉重。
- **泛化性限制**：仅在单一室内数据集测试，未在真实机器人或更多样化场景（户外、动态物体）中验证。
