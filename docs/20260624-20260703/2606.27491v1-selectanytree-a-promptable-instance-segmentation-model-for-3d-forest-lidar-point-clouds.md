## 摘要
随着森林监测向可扩展、精细化的三维测量发展，森林LiDAR点云的自动实例分割变得日益关键。然而，树实例的标签稀缺制约了进展：单公顷可能包含数百万个点和数百个重叠、复杂的树冠，从原始数据中手动从头进行标注既费力又容易出错。标注通常从自动预分割结果中修正而来，但由于无法提供交互式或AI辅助的精细化处理，成本依然高昂。受基础分割模型的可提示范式启发，我们提出了SelectAnyTree，一种可提示的实例分割模型，只需几次点击即可在3D森林点云中描绘出任意单棵树。它引入了两个关键组件：点击到查询的提示编码器（Click-to-query prompt encoder）和冠层高度模型（CHM）引导的初始提示。前者将每次点击转化为单个内容查询，编码其3D位置、正/负极性以及池化的局部骨干特征。后者在无需任何用户输入的情况下，提供树顶作为几何和生态引导的初始提示。随后，状态空间查询解码器将得到的提示查询解码为单个树掩膜，以线性时间复杂度在大规模森林场景中高效捕捉长程上下文。我们在七个不同森林区域和一个独立的留出测试数据集上，在交互式和实例级设置下评估了SelectAnyTree，展示了超越训练域的强泛化能力。它通过单次点击即可分割目标树，交并比（IoU）达到78.2，比最强的可提示基线高出24.8个百分点，并以最少的点击达到每个精度目标，同时使用的参数和推理时间远少于先前的可提示模型。源代码可在https://github.com/thanhhff/SelectAnyTree获取。

## Abstract
Automated instance segmentation of forest LiDAR point clouds is increasingly critical as forest monitoring moves toward scalable, detailed, 3D measurement. Yet, progress is constrained by label scarcity for tree instances; a single hectare can hold millions of points and hundreds of overlapping, complex crowns, making manual annotation from scratch with raw data laborious and error-prone. Annotations are often corrected from automatic pre-segmentations, but remain costly as these provide no interactive or AI-assisted refinement. Inspired by the promptable paradigm of foundation segmentation models, we propose SelectAnyTree, a promptable instance segmentation model that delineates any individual tree in a 3D forest point cloud from a few clicks. It introduces two key components: Click-to-query prompt encoder and Canopy Height Model (CHM)-guided first prompt. The former turns each click into a single content query, encoding its 3D position and positive/negative polarity together with a pooled local backbone feature. The latter provides treetops as a geometry- and ecologically guided first prompt without any user input. The resulting prompt query is then decoded into one tree mask by a state-space query decoder to efficiently capture long-range context in large-scale forest scenes with linear-time complexity. We evaluate SelectAnyTree in interactive and instance-level settings across seven diverse forest regions and an independent held-out test dataset, demonstrating strong generalization beyond the training domains. It segments a target tree to 78.2 Intersection over Union (IoU) from a single click, 24.8 points above the strongest promptable baseline, and reaches every accuracy target with the fewest clicks, while using far fewer parameters and less inference time than prior promptable models. The source code is available at https://github.com/thanhhff/SelectAnyTree.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- 设计一个**可提示的实例分割模型**，该模型将森林场景一次编码，然后根据用户提供的点击（正/负）生成单个树掩膜；引入森林特有的几何先验（CHM树顶）作为免费初始提示，提升首次点击的精度；并通过共享的点击模拟迭代策略（training/evaluation rollout）保证训练与测试的一致性。

### 关键技术细节
1. **3D场景编码器**  
   - 将输入点云体素化（体素分辨率0.2m），使用稀疏编码器（基于Mamba的状态空间模型）产生每体素特征，并缓存。后续所有提示共享该特征，实现高效多树分割。  
   - 附加辅助分支：预测每体素树/非树二分类和判别性嵌入，用于保持骨干网络健康。

2. **提示编码器（Prompt Encoder）**  
   - 将一组点击转换为单个内容查询（query）q，该查询位于与骨干网络相同的特征空间。  
   - **几何锚点**：在正点击水平半径1.5m圆柱体内池化骨干特征，捕获局部冠层上下文。  
   - **位置与标签编码**：点击坐标使用随机傅里叶位置编码；点击极性（正/负）通过可学习嵌入映射；二者求和后经MLP映射到骨干宽度。  
   - **带符号聚合**：正点击取+1、负点击取-1，进行带符号均值聚合，使查询向目标拉近、远离非目标。  
   - **最终查询**：聚合令牌与几何锚点残差相加，通过线性投影得到q。

3. **CHM引导的首次提示**  
   - 利用冠层高度模型（CHM）自动检测每棵树的树顶（最高点）作为正点击，并结合树体内部最近质心的点作为初始正点击。该过程无需用户输入，视为免费提示。

4. **查询解码器（Query Decoder）**  
   - 使用基于状态空间模型（Mamba）的轻量级解码器，通过局部k近邻聚合和状态空间建模迭代精炼查询，输出体素级别的掩膜logits和对象性分数。  
   - 线性时间复杂度，适合大规模森林场景。

5. **交互式点击模拟迭代（Interactive Rollout）**  
   - 训练与评估使用完全相同的模拟过程：从首次提示开始，解码当前掩膜，对比地面真值，从更大的错误区域（假阴性或假阳性）中采样一个修正点击（正或负），追加到点击集，重复直到达到预设迭代次数（训练设为5次）。  
   - 该策略确保优化目标与评估协议完全一致。

### 训练损失
- 总损失 L = λp * L_mask + λbin * L_bin + λdis * L_dis  
  - L_mask：掩膜损失（BCE + Dice，λdice=2.0），在每个迭代步骤均施加监督。  
  - L_bin：辅助树/非树交叉熵损失。  
  - L_dis：判别性嵌入损失（拉近同类体素、推远不同类）。  
  - 权重均为1.0。

## 7. 优点：方法或实验设计上的亮点

- **森林感知的可提示设计**：首次将CHM树顶作为几何引导的免费提示，与点击提示融合，大幅提升首次点击精度，且不增加用户负担。  
- **统一的点击模拟迭代**：训练和评估使用完全相同的模拟流程，避免了训练-评估不匹配问题，使得模型直接优化测试时交互行为。  
- **高效的状态空间解码器**：线性复杂度，支持大规模森林场景的实时交互。  
- **场景特征缓存复用**：一次编码即可响应多个提示，支持多树并行高效分割。  
- **全面的公平性验证**：不仅给出了消融，还额外为所有基线加上CHM提示，证明CHM本身不是优势的唯一来源，排除了不公平对比（表7）。  
- **跨域泛化测试**：使用完全独立的LAUTx数据集，且未进行任何微调，验证模型实际通用能力。

## 8. 不足与局限

- **CHM依赖性**：自首次提示依靠CHM，若CHM在密集、多层冠层中预测不可靠（如体素分类错误），则性能可能下降。在LAUTx上无CHM变体反而更好，说明固定异速参数（α,β）和最小高度阈值等超参数可能不适应所有森林类型。

- **零点击自动分割表现有限**：在自动全场景分割任务上远不如专用模型（虽然并非设计初衷），限制了其作为完全自动工具的使用场景。  
- **未探索更丰富的提示类型**：如边界框、涂鸦、多假设掩膜等，未来可扩展。
