## 摘要
一个3D场景是通过其对象来理解的，而不是组成对象的基元。然而，前馈重建方法输出密集的、无结构的点集或高斯集合，对象级别的结构需要事后恢复。我们提出了一种前馈框架，直接从无位姿的多视图图像中将场景分解为实例结构的3D标记组——紧凑的以对象为中心的单位，重建、分割和操作都由此而来。每个标记组将捕获实体级身份的实例标记与编码局部几何和外观的锚点标记配对，这些标记被解码为一组3D高斯。这种两级分解将对象身份与局部外观解耦，使对象实例成为表示的原生接口而非派生产物。标记组通过可微渲染学习，联合重建和分割监督，无需3D注释。我们的前馈模型在类别无关的实例分割上超越了每个场景优化的基线，同时在新视图合成上保持竞争力。除了这些指标，相同的标记组直接实现了实例级场景编辑——通过对其组进行操作来移除、平移或插入对象——以及高效的开词汇3D实例检索，其中检索复杂度随实例数量而非基元数量缩放。

## Abstract
A 3D scene is understood through its objects, not the primitives that compose them. Yet feed-forward reconstruction methods output dense, unstructured sets of points or Gaussians, leaving object-level structure to be recovered after the fact. We propose a feed-forward framework that decomposes a scene into instance-structured 3D token groups directly from unposed multi-view images -- compact object-centric units from which reconstruction, segmentation, and manipulation all follow. Each token group pairs an instance token capturing entity-level identity with anchor tokens that encode local geometry and appearance, which are decoded into a set of 3D Gaussians. This two-level factorization decouples object identity from local appearance, making object instances a native interface of the representation rather than a derived product. The token groups are learned through differentiable rendering with joint reconstruction and segmentation supervision, requiring no 3D annotations. Our feed-forward model surpasses per-scene optimization baselines in class-agnostic instance segmentation while remaining competitive in novel view synthesis. Beyond these metrics, the same token groups directly unlock instance-level scene editing -- removing, translating, or inserting objects by operating on their groups -- as well as efficient open-vocabulary 3D instance retrieval, where retrieval complexity scales with the number of instances rather than primitives.

### 论文核心问题与整体含义（研究动机和背景）

- **核心问题**：现有前馈式3D重建方法（如pixelSplat、MVSplat、VGGT等）输出密集、无结构的点云或3D高斯集合，缺乏对场景中物体实例的显式建模。对象级理解（如分割、编辑、检索）必须通过后处理从基元中恢复，导致效率低下且不自然。
- **根本矛盾**：人类理解场景是通过物体（对象），而非组成物体的基元。当前表示以基元为单位，对象身份信息分散在大量基元中，无法直接作为统一操作接口。
- **论文目标**：构建一种**前馈式、从无位姿多视图图像直接生成的实例结构化3D表示**，使对象实例成为表示的原生单元，同时保留细粒度的几何与外观细节，从而无缝支持重建、分割、编辑与检索。

### 方法论：核心思想、关键技术细节

- **核心思想**：将场景分解为一系列**实例结构化的3D标记组（Token Groups）**。每个组包含：
  - **实例标记（Instance Token）**：编码整个物体实体的身份与范围（全局、语义级）。
  - **一组锚点标记（Anchor Tokens）**：编码局部几何与外观（精细级），每个锚点解码为多个3D高斯。
- **两级分解**：实例标记与锚点标记分离，使对象身份与局部细节解耦，对象成为可操作的第一公民。
- **技术流程**：
  1. **多视图上下文编码**：使用冻结的3D基础模型（VGGT）提取多视图特征和点图，下采样后与图像、点图特征融合，形成上下文标记。
  2. **锚点标记生成**：通过最远点采样选择锚点位置，用图像-锚点Transformer（🏷️Danchor）交叉关注上下文，初始化解码为锚点标记。
  3. **实例标记生成**：使用锚点-分组Transformer（🏷️Dgroup）将锚点标记作为键/值，查询学习到的分组标记，得到实例标记。
  4. **锚点-实例分配**：实例标记与锚点标记计算点积相似度，经softmax（含void通道）分配每个锚点属于哪个实例，形成硬隶属关系。
  5. **高斯重建**：每个锚点标记通过MLP预测多个3D高斯（位置偏移、尺度、旋转、不透明度、球谐），所有高斯继承其父锚点的实例分配。
  6. **训练损失**：
     - 渲染损失：L_render = MSE + λ_lpips LPIPS，监督锚点token的质量。
     - 分割损失：通过渲染实例概率图，与2D实例掩码进行匈牙利匹配，计算Dice+BCE损失L_seg，监督实例token的分组。
     - 联合损失：L = L_render + λ_seg L_seg，使用线性warm-up逐步激活L_seg。
  7. **语义特征蒸馏（可选）**：将2D基础模型特征（LSeg）蒸馏到标记组中，分解为**组级共享嵌入**（高维）和**锚点级低维残差**，渲染时组合得到每像素语义特征，训练使用余弦相似度损失和组级对齐损失。
- **公式/文字流程**：多视图特征融合 → 上下文标记 → Danchor解码锚点标记 → Dgroup生成分组标记 → softmax分配 → 高斯预测 → 可微渲染 → 联合优化。

### 主要结论与发现

- 实例结构化的token组能够**直接从无位姿多视图图像中前馈式学习**，无需3D标注或每场景优化。
- 在类无关实例分割上**超越所有对比的每场景优化方法**（AP 0.235 vs. 0.178/0.139），证明了原生实例结构的优势。
- 新视图合成质量**接近但略低于像素对齐方法**，但特征存储量从百万级降至数万级，且在零样本泛化上表现更优（MipNeRF360 PSNR 16.52 vs. 14.58）。
- **联合训练**是必要的：顺序训练分割AP仅0.032；warm-up稳定训练，移除后AP降至0.081。
- **分解式特征蒸馏**有效：组级嵌入承担主要语义，锚点残差捕捉细部变化，完整分解优于单一组件。
- 同一token组直接支持**实例级编辑和开放词汇检索**，无需后处理模块。

### 优点（方法或实验设计亮点）

1. **创新表示机制**：将对象实例作为原生接口，而非事后聚合，概念简洁且具有泛化潜力。
2. **完全前馈**：单次前向传播即可获得场景的实例+几何+外观表示，无需COLMAP、恢复位姿或逐场景优化。
3. **无3D标注学习**：仅依赖2D RGB和实例掩码监督，实例结构从2D掩码中自动涌现。
4. **特征存储极高效**：将高维语义特征分解为组级共享+锚点低维残差，存储量比Uni3R低两个数量级。
5. **多任务自然统一**：重建、分割、编辑、检索共享同一表示，无需额外组件。
6. **实验覆盖全面**：不仅在标准指标上比较，还提供了泛化实验和消融，增强了结论可靠性。

### 不足与局限

1. **场景规模与类型限制**：目前仅在ScanNet室内场景做主要评估（最大实例数固定为100），未验证在大型无界场景（如室外街景、复杂室内）中的表现。
2. **重建质量略逊**：由于使用紧凑token组而非全像素对齐，PSNR/SSIM低于Uni3R等（但零样本泛化更好），在精细纹理重建上有损失。
3. **编辑鲁棒性未充分验证**：仅展示分离良好物体上的编辑，遮挡、接触、部分被遮挡物体的操作效果未量化评估。
4. **语义检索无定量指标**：开放词汇检索仅有定性示例，缺乏mAP等定量对比。
5. **动态场景不可用**：当前框架针对静态场景，无法处理运动物体。
6. **锚点数量固定**：K=1024锚点，L=100组，可能不适用于高度复杂或多实例场景。
7. **依赖2D分割质量**：训练时使用2D实例掩码（ScanNet人工标注或SAM2伪标签），噪声大小影响分组质量，但实验表明可容忍一定噪声（RealEstate10K）。
