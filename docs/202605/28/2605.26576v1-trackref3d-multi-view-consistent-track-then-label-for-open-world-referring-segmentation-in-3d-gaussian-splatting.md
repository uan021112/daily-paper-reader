## 摘要
基于3D高斯泼溅的指代表达分割（R3DGS）利用自然语言进行3D物体分割，已成为具身人工智能的一项关键能力。然而，现有方法通常依赖昂贵的逐场景手动标注和逐视图伪掩码生成，这导致了多视图不一致以及对不同查询特异性的泛化能力差。为了解决这一问题，我们提出了TrackRef3D，一个全自动流水线，通过引入一种多视图一致的跟踪-标注范式，在3D高斯泼溅（3DGS）中实现了无需人工标注的开放世界指代表达分割，该范式从根本上将物体发现与语义定位解耦。具体而言，我们提出了一个轨迹感知语义共识模块（TSCM），该模块通过同义词聚类和轨迹感知投票聚合跨视图预测，以建立规范的语义身份，从而确保多视图一致性。此外，我们采用了一种可见性感知的描述生成策略来减少歧义，并提出了一种混合训练策略（HTS），该策略联合优化粗粒度类别语义和细粒度指征线索，通过多正例对比目标确保在不同查询特异性下的鲁棒性。在基准测试上的大量实验表明，TrackRef3D达到了最先进的性能。

## Abstract
Referring 3D Gaussian Splatting (R3DGS), which utilizes natural language for 3D object segmentation, has emerged as a crucial capability for embodied AI. However, existing methods typically rely on expensive per-scene manual annotation and per-view pseudo mask generation, which suffer from multi-view inconsistency and poor generalization to varying query specificities. To address this, we present TrackRef3D, a fully automatic pipeline that achieves open-world referring segmentation in 3D Gaussian Splatting (3DGS) without manual annotation by introducing a multi-view consistent track-then-label paradigm that fundamentally decouples object discovery from semantic grounding. Specifically, we propose a Trajectory-Aware Semantic Consensus Module (TSCM) which aggregates cross-view predictions via synonymous clustering and trajectory-aware voting to establish a canonical semantic identity, thereby ensuring multi-view consistency. Furthermore, we employ a visibility-aware description generation strategy to mitigate ambiguity and propose a Hybrid Training Strategy (HTS) that jointly optimizes coarse category semantics and fine-grained referential cues to ensure robustness under varying query specificities using a multi-positive contrastive objective. Extensive experiments on benchmarks demonstrate that TrackRef3D achieves state-of-the-art performance.

- **现有问题**：
  - 现有方法（如ReferSplat）依赖昂贵的**逐场景人工标注**（类别标签、指代描述）和**逐视图生成伪掩码**，成本高昂，难以扩展至真实部署。
  - 逐视图伪掩码缺乏跨视图一致性，当物体出现遮挡或视角变化时，同一物体在不同视图下可能被生成不一致的掩码和标签，导致**多视图不一致**。
  - 训练时过度拟合长描述，导致模型对用户实际常用的**短查询泛化能力差**。
- **研究目标**：提出一种**全自动、无需人工标注的开放世界**指代分割流水线，同时解决多视图不一致和不同查询特异性下的鲁棒性问题。

#### 2. 方法论：核心思想、关键技术细节

- **核心思想**：引入**多视图一致的“跟踪-标注”（Track-then-Label）范式**，将物体发现与语义定位**完全解耦**。首先通过视频跟踪获得稳定轨迹，再基于轨迹进行语义共识和描述生成。
- **关键技术细节**：
  1. **初始物体发现**：使用2D基础模型（Florence-2 + SAM-2）逐视图生成物体掩码和开放词汇类别标签。
  2. **轨迹感知语义共识模块（TSCM）**：
     - **视频跟踪**：使用DEVA将逐视图掩码关联为时间上一致的轨迹（唯一track ID），抵御遮挡和视角变化。
     - **同义词聚类**：对同一轨迹中不同视图预测的类别标签进行语义嵌入余弦相似度聚类（阈值τ_sem=0.85），合并语义等价的标签（如“ramen”和“food”）。
     - **轨迹感知投票**：统计每个轨迹中各类别出现频率，选出最频繁的作为该轨迹的**规范语义身份**（canonical semantic identity）并传播到所有视图，确保跨视图一致。
     - **可见性感知描述生成**：计算每个视图的可见性分数（公式2），选择物体大小适中、空间上下文清晰的**关键帧**，利用视觉语言模型（CogVLM-2）生成多粒度描述，减少遮挡造成的歧义。
  3. **混合训练策略（HTS）**：
     - 将规范语义身份（短类别词）和关键帧生成的描述（长描述）均视为同一轨迹的**正样本**。
     - 使用**多正例对比损失**（公式4）联合优化：使高斯特征嵌入与所有正样本文本嵌入对齐，同时与负样本（其他物体）分离。
     - 总损失：`L = L_seg + λ L_con`，其中`L_seg`为二值交叉熵分割损失，`L_con`为对比损失（λ=0.1）。
- **算法流程**：输入多视图RGB图像 → 重建3DGS几何 → 初始物体发现 → TSCM（跟踪→聚类→投票→生成描述）→ HTS训练语言引导的3D高斯指代字段 → 推理时根据任意描述渲染分割掩码。
