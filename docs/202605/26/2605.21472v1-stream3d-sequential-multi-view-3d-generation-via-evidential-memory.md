## 摘要
基于视图条件的3D生成器（如SAM 3D、TRELLIS和Hunyuan3D）能够从单视图生成高质量的对象重建，但现实世界的视觉观察通常以长单目流的形式出现。将这些生成器独立应用于每个流式帧会导致生成结果严重的时序不一致性。为解决此问题，我们提出Stream3D，这是首个无需训练的流式机制，它将冻结的视图条件3D生成器转变为具有恒定跨块记忆的流式生成器。Stream3D通过维护一个紧凑的证据记忆来实现这一点，该记忆基于提出的证据分数机制选择性缓存最具信息量的历史帧。随着流的推进，记忆动态更新以保留固定数量的信息帧，防止记忆占用随序列长度线性增长。这还防止了长序列上的退化，并保持底层生成器完全不变，无需重新训练、架构修改或辅助损失。在真实和合成流式基准上的评估表明，Stream3D在光度指标和几何指标上均优于潜在传输基线方法，包括KV缓存复用和基于流的特征编辑。更多详情请见：https://anonymous-submission-20.github.io/streaming3D.github.io/。

## Abstract
View-conditioned 3D generators such as SAM 3D, TRELLIS and Hunyuan3D produce high-quality object reconstructions from a single view, but real-world visual observation often arrives as long monocular streams. Naively applying these generators to each streaming frame independently leads to severe temporal inconsistency in the generated results. To address this problem, we propose Stream3D, the first training-free streaming mechanism that turns a frozen view-conditioned 3D generator into a streaming generator with constant cross-chunk memory. Stream3D achieves this by maintaining a compact evidential memory, which selectively caches the most informative historical frames based on a proposed evidence score mechanism. As the stream progresses, the memory dynamically updates to retain a fixed number of informative frames, preventing the memory footprint from growing linearly with sequence length. This also prevents degradation over long sequences and keeps the underlying generator completely unchanged without retraining, architectural modifications, or auxiliary losses. Evaluated on both realistic and synthetic streaming benchmarks, Stream3D outperforms latent-transport baselines, including KV-cache reuse and flow-based feature editing, across both photometric and geometric metrics. More details can be found at: https://anonymous-submission-20.github.io/streaming3D.github.io/.

## 2. 方法论

### 核心思想
放弃传输潜在状态，转而维护一个紧凑的**证据记忆（Evidential Memory）**：根据生成器自身的跨注意力（cross-attention）图，识别对每个3D查询令牌（query token）最可靠的**历史视图**，并动态保留固定数量的高证据帧索引。在生成时，每个令牌可以检索其最信任的历史观测，实现细粒度的多视图条件融合。

### 关键技术细节

1. **证据分数（Evidence Score）**
   - 在每个块（chunk）处理时，先运行**一步预热去噪**（使用跨块固定的初始噪声z₀），提取生成器结构阶段某层（L）的跨注意力图Aₐ。
   - 对每个查询令牌q和每个输入视图v，计算归一化注意力分布，然后计算**熵**Hₐ[q]（衡量注意力选择的明确性）。
   - 证据分数Mₐ[q]定义为：结合该视图的总注意力强度与注意力集中度（1 - 熵），最终归一化到[0,1]。高分数表示该视图对该令牌既被“强烈关注”又“选择明确”。

2. **自适应证据记忆（Adaptive Evidential Memory）**
   - 维护两个矩阵：**证据分数矩阵M**（Q×D，Q为查询令牌数，D为每个令牌保留的候选帧数）和**帧索引矩阵F**（Q×D）。
   - 每处理一个新块，对于每个令牌，将当前块中各视图的证据分数与历史记忆中的前D个分数合并，**保留最高的D个**。因此内存占用始终为2×Q×D标量（例如SAM 3D中Q=4096, D=5时仅约65KB）。
   - 记忆仅存储帧索引和分数，不存储任何潜在状态，**不随序列长度增长**。

3. **基于证据的多视图生成（Evidence-Based Multi-Generation）**
   - 在每个块的全生成过程前，根据记忆矩阵F统计每个历史帧被多少（令牌，排名）槽位占据（即所有权计数n_fr），选择**Top-K个帧**（K为捆绑大小）作为条件视图集V*。
   - 在生成器（fθ）的多视图扩散过程中，对每个查询令牌q，使用证据分数作为**视图加权融合系数**（公式5），融合各视图的预测速度。
   - 全生成使用与预热相同的固定噪声z₀（但每块可重新采样z_k），最终输出3D表示（高斯泼溅或网格）。

### 算法流程（文字描述）

1. 初始化：固定噪声z₀，清空证据记忆M、F；
2. 对每个块k：
   - 运行一步预热去噪，提取跨注意力，计算块内各视图的证据分数；
   - 更新记忆：每个令牌合并新旧证据分数，保留Top-D；
   - 根据所有权计数选定Top-K视图作为条件束；
   - 用这些视图进行完全的多视图生成（融合加权），输出本块的3D样本。

### 三个关键设计
- **冻结的初始噪声z₀**：跨块复用使证据分数可比较；
- **仅一步预热**：因为第一步注意力主要由输入视图内容主导，后续步骤受模型自身预测影响，不适宜做视图选择；
- **记忆深度D与捆绑大小K分离**：D控制证据计算的鲁棒性，K控制生成器实际使用的视图数。

## 6. 主要结论与发现

- Stream3D在GSO和NAVI上**在所有指标上均优于**单视图生成器、多视图扩散融合基线以及流式替代方案（KV缓存、FlowEdit、MV-SAM3D）。
- 与潜在传输方法相比，Stream3D**避免了长序列退化**，因为其证据记忆具有**非递减性质**（证据分数只增不减），而KV缓存和FlowEdit随序列长度性能下降。
- 消融证实：在更深的阶段提取证据分数效果更好；**归一化的证据分数**优于熵或未归一化版本；记忆深度D取3~5时平衡性能与成本；捆绑大小K=8或16足够，更多视图提升有限。
- 核心主张得到验证：流式3D生成应视为**证据选择问题**而非潜在传输问题；保留令牌级证据视图更稳定、可扩展。

## 7. 优点

- **训练-free**：不修改生成器权重、架构，无需额外损失或重训练，可直接包装任意视图条件生成器（论文声称，但仅验证了SAM3D）。
- **内存恒定**：记忆大小为2×Q×D，不随流长度增长，适用于无限长视频流。
- **可解释性**：证据分数基于跨注意力，可理解每个令牌依赖哪些历史视图。
- **非退化性质**：证据分数单调非减，生成质量在流推进中只提升不下降（理论上），实际结果也避免了长序列误差累积。
- **计算高效**：每个块仅需一次额外预热前向，开销较小。
- **方法简单**：只需三个超参数（L,D,K），无需学习。

## 8. 不足与局限

- **依赖基础生成器质量**：如果底层单视图生成器本身无法从单个视图重建出合理几何或纹理，那么证据记忆也无法完全恢复。例如，在剧烈遮挡或视角缺失严重的区域，基模型的失败无法通过记忆弥补。
- **仅验证了一种骨干**：论文实验全部基于SAM3D，未展示在TRELLIS、Hunyuan3D等生成器上的通用性。理论上声称可扩展，但缺乏实证。
- **相机姿态依赖**：使用Depth Anything 3估计姿态和深度，若姿态估计有误差，可能影响视图对齐和证据可靠性。论文未分析此误差的影响。
- **实验场景有限**：评估仅在物体级别的合成和真实数据集上进行，未涉及更复杂的场景级流式3D生成（如室内环境）。
- **超参数选择**：默认D=1、K=8，消融表明更大D/K可进一步提升，但未给出自动选择策略。实际应用可能需要针对不同场景调整。
- **与最新流式重建方法的比较缺失**：论文未与Stream3R、VGGT-SLAM等端到端流式重建方法对比（尽管任务目标不同——生成 vs. 重建），但可额外讨论其差异。总体而言，局限被作者在原文中部分承认（第5节）。
