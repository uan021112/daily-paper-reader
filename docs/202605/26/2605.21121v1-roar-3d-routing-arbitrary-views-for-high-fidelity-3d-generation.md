## 摘要
单图到3D生成模型现在可以产生高质量的几何形状，但由于仅基于单一视图进行条件生成，不可避免地会在未观察区域引入歧义。多视图条件生成可以减少这种歧义，但现有方法要么需要固定的规范视角，要么依赖外部重建模块，这会带来沉重的训练成本并限制生成质量。我们观察到预训练的单视图模型已经具备强大的2D到3D基础，可以重用于多视图条件生成。然而，深入分析表明，其条件生成机制将方向控制与几何传递纠缠在一起，当来自不同视角的图像简单组合时，这两个功能会发生冲突。基于这一分析，我们提出ROAR-3D，一种轻量级方法，将预训练的单视图模型升级为接受任意数量的无位姿图像。一个逐token的视图路由器将每个3D潜在token分配给与其最相关的视图，无需显式位姿输入即可隐式建立2D到3D对应关系。一种双流注意力设计保留了预训练的主视图行为，同时通过专门用于几何增强的独立路径路由辅助视图。一种方向扰动策略确保辅助路径学习与方向无关的几何传递。这些组件引入了最少的可训练参数，并且相对于单视图基线增加了可忽略的推理开销。ROAR-3D实现了最先进的多视图3D生成质量，并支持从1到12+视图的测试时视图缩放，持续改进。

## Abstract
Single-image-to-3D generative models can now produce high-quality geometry, yet conditioning on a single view inevitably introduces ambiguity about unseen regions. Multi-view conditioning can reduce this ambiguity, but existing methods either require fixed canonical viewpoints or rely on external reconstruction modules that impose heavy training costs and limit generation quality. We observe that pretrained single-view models already possess strong 2D-to-3D grounding that can be reused for multi-view conditioning. However, a closer analysis reveals that their conditioning mechanism entangles orientation control with geometry transfer, two functions that conflict when images from different viewpoints are naively combined. Based on this analysis, we propose ROAR-3D, a lightweight method that upgrades a pretrained single-view model to accept an arbitrary number of unposed images. A token-wise view router assigns each 3D latent token to its most relevant view, implicitly establishing 2D-to-3D correspondences without explicit pose input. A dual-stream attention design preserves the pretrained primary-view behavior while routing auxiliary views through a separate path dedicated to geometric enrichment. An orientation perturbation strategy ensures the auxiliary path learns orientation-independent geometry transfer. These components introduce minimal trainable parameters and add negligible inference overhead relative to the single-view baseline. ROAR-3D achieves state-of-the-art multi-view 3D generation quality and supports test-time view scaling from 1 to 12+ views with consistent improvements.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：单图像到 3D 生成模型虽然能生成高质量几何，但仅依赖单一视图会引入“未见区域”的歧义（模型必须幻觉未观察到的结构）。多视图条件生成可以减少这种歧义，但现有方法存在两大局限：
  - 第一类要求固定规范视角（如预定义的前、后、左、右），无法处理无约束的真实输入。
  - 第二类依赖外部重建模块（如 VGGT）将特征注入生成管线，存在域差距且生成质量受限于重建精度，训练成本高。
- **关键观察**：预训练的单视图 3D 生成模型已经具备强大的 2D→3D 对应关系知识（其交叉注意力层隐含学习了不同视角下的 2D-3D 对应）。这些层是最昂贵的部分，应被复用，而非从头重新学习。
- **挑战**：单视图模型中，条件机制将“方向控制”与“几何结构传递”纠缠在一起。多视图下，不同视角的方向信号冲突，导致生成不稳定。
- **整体含义**：本文提出一种轻量级方法 ROAR-3D，在不重训骨干网络、不引入外部编码器、不依赖相机位姿的前提下，将预训练单视图模型升级为可接受任意数量无位姿图像输入，实现高质量多视图条件 3D 生成。

## 2. 方法论

### 核心思想
- 通过**逐 token 视图路由器**将每个 3D 潜在 token 分配给最相关的输入视图，隐式建立 2D-3D 对应关系；利用**双流交叉注意力**分离主视图（设定方向和基础几何）与辅助视图（仅贡献几何细节）；采用**方向扰动策略**强制辅助流学习方向无关的几何传递。

### 关键技术细节

1. **Token-Wise View Router（逐 token 视图路由器）**
   - 在每个 Transformer block 中独立运行。
   - 3D 潜在 token Z ∈ R^(N×D) 与 V 个输入视图的 DINOv2 patch 特征 F_v ∈ R^(S×D) 交互。
   - 从预训练交叉注意力权重初始化查询投影 W_q、键投影 W_k。
   - 对每个 3D token i，计算其与每个视图 v 的路由 logit：r_i,v = w_agg^T · [head-attention scores]。使用 Gumbel-Softmax 进行可微分的离散选择，选中最匹配的视图 v*_i。
   - 推理时无噪声，选择是确定性的。

2. **Dual-Stream Cross-Attention（双流交叉注意力）**
   - 包含两个结构相同的交叉注意力模块：主流 CA_p（继承预训练权重）、辅助流 CA_a（复制 CA_p 参数）。
   - 指定一个视图为主视图 v_p。路由器分配给主视图的 token 由 CA_p 处理，其他 token 由 CA_a 处理。
   - 主流保持预训练的双重角色（方向+几何），辅助流**只负责几何结构传递**，不改变方向。
   - 输出通过软权重 y_i 调制以传递梯度。

3. **Orientation Perturbation（方向扰动）**
   - 阻止辅助流 CA_a 利用规范对齐的捷径，强制其学习方向无关的几何映射。
   - 训练时以概率 p_pert = 0.2 同时执行：
     - (i) 将 3D 点云随机旋转（0/90/180/270°），且避免与任何输入视图对齐。
     - (ii) 移除主视图指定：所有视图均视为辅助，全部由 CA_a 处理。
     - (iii) 仅更新 CA_a，冻结 CA_p。
   - 该策略使 CA_a 必须仅依赖内容特征建立 2D-3D 对应，独立于方向先验。

4. **几何细化阶段**
   - 基于 LATTICE 的第二阶段模型，在更高分辨率潜在空间上采用相同的多视图条件结构（路由器、双流注意力、方向扰动），以生成更精细的几何细节。

- **整体流程**：给定任意数量无位姿图像 → DINOv2 提取 patch 特征 → 每个 Transformer block 中路由器分配 token → 双流交叉注意力获取融合特征 → 扩散 Transformer 生成 3D 潜在 → VAE 解码为粗几何 → 细化阶段提升细节。

## 6. 主要结论与发现

1. **ROAR-3D 在多视图 3D 生成上达到 SOTA**：第一阶段的 CD 比最强基线 ReconViaGen 降低 21%（25.372 vs 32.055），F1(0.05) 提高 5.6%。第二阶段进一步将 CD 降至 21.039，F1(0.05) 提升至 81.6%。

3. **支持测试时可扩展**：训练时 1–4 视图，推理时稳定推广至 12+ 视图，且性能持续提升。
4. **组件有效性**：路由器解决视图选择歧义，双流注意力分离方向/几何，方向扰动阻止捷径学习，三者必不可少。
5. **对真实世界输入鲁棒**：在 Anyview-200 上优于所有基线，能处理局部/特写等 OOD 输入。

## 7. 优点

- **轻量高效**：仅需少量可训练参数（路由器+辅助交叉注意力），训练成本低（3–4 GPU days），推理无额外开销。
