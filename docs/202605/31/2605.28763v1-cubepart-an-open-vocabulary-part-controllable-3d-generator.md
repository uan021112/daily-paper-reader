## 摘要
交互式3D资产在游戏和仿真中通常被分解为特定的语义部件以支持动画、物理和脚本行为，然而大多数生成式3D模型要么生成整体网格，要么生成任意的部件分解，无法与应用特定需求对齐。我们提出CubePart，一个用于开放词汇、部件可控的3D网格生成的生成框架，将部件结构作为显式的推理时控制信号。给定一个全局文本提示和一个用户定义的部件模式（以开放式部件名称列表表示），我们的方法生成一组网格——每个模式元素一个——这些网格在遵循指定语义结构的同时组装成一个连贯的对象。为了实现这一能力，我们引入了一个可扩展的数据管道来构建一个大型的开放词汇、部件标注的3D数据集，以及一个将全局形状合成与部件级解码分离的两阶段生成架构。我们证明，所得资产可以直接集成到游戏引擎中，并由动画和行为脚本驱动，无需手动后处理。项目页面：https://cubepart.github.io/

## Abstract
Interactive 3D assets used in games and simulation are typically decomposed into specific semantic parts to support animation, physics, and scripted behaviors, yet most generative 3D models produce either monolithic meshes or arbitrary part decompositions that cannot be aligned with application-specific requirements. We present CubePart, a generative framework for open-vocabulary, part-controllable 3D mesh generation that exposes part structure as an explicit inference-time control signal. Given a global text prompt and a user-defined parts schema expressed as an open-ended list of part names, our method generates a set of meshes - one per schema element - that assemble into a coherent object while respecting the specified semantic structure. To enable this capability, we introduce a scalable data pipeline to construct a large open-vocabulary, part-labeled 3D dataset, along with a two-stage generative architecture that separates global shape synthesis from part-level decoding. We demonstrate that the resulting assets can be directly integrated into game engines and driven by animation and behavior scripts without manual post-processing. ## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现有3D生成模型（如DreamFusion、3DShape2VecSet、Trellis等）普遍输出**整体网格**或**任意分割**，缺乏对语义部件结构的显式控制，导致生成的资产无法直接用于游戏、仿真等需要按特定部件分解以实现动画、物理和脚本行为的场景中。用户无法指定部件数量、名称和语义，导致资产需要大量人工后处理。
- **整体含义**：游戏开发者需要资产按照特定“模式”（如汽车必须有四个车轮和一个底盘）分解，而现有方法无法满足这种应用特定需求。本文提出CubePart，首次允许用户通过**开放词汇的文本部件模式**作为推理时控制信号，生成符合指定部件结构的3D网格集合，直接集成到游戏引擎中，无需手动后处理。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：CubePart采用**两阶段生成架构**：
  - **第一阶段（单网格生成）**：基于文本提示和部件模式，生成整体3D网格（单网格）。
  - **第二阶段（多网格生成）**：将整体网格分解为对应部件模式的多个独立网格，同时保持全局几何连贯性。
- **关键技术细节**：
  - **VecSet扩散模型**：基于3DShape2VecSet的变分自编码器（VAE）将3D网格编码为潜向量集，使用多模态扩散Transformer（MM-DiT）进行流匹配生成。
  - **模式感知微调**：在Stage1预训练模型上，使用结构化提示（包含全局描述+部件列表）进行微调，确保生成网格包含所有指定部件。
  - **交叉部件注意力块**：Stage2中插入**零初始化Transformer块**，实现部件间的全局信息交换，同时不破坏预训练的单网格生成先验。这些块仅插入第1、5、9、17层，相比修改所有现有层更高效。
  - **部件感知提示**：每个部件的生成条件为“此对象包含以下部件：{所有部件列表}。目标分割：{目标部件名}”，提供全局上下文，帮助模型理解边界。

## 6. 主要结论与发现

- CubePart是**首个**实现开放词汇、部件可控的3D网格生成框架，用户可通过文本模式指定任意部件分解。
- 两阶段架构有效分离全局形状合成与部件分解，交叉部件注意力块在保持预训练先验的同时实现高效部件间通信。
- 自动化数据管道（VLM+Set-of-Mark）可大规模构建高质量部件标注数据集（462K资产/2M部件），远超现有数据集。
- 生成的多部件网格可直接用于游戏引擎中的焊接、铰链、物理力控制和交互脚本，无需人工后处理。
- 消融实验表明：预训练、模式微调、交叉注意力均显著提升生成质量，且零初始化交叉注意力优于全局修改现有层的方法。

## 7. 优点

- **方法创新**：将开放词汇的文本部件模式作为显式控制信号，解决了现有方法无法按用户需求指定部件结构的问题。
- **两阶段架构**：先整体后分解，利用预训练先验，降低部件生成难度。
- **零初始化交叉注意力**：最小化对预训练能力的破坏，高效实现部件间全局交互。
- **可扩展数据管道**：完全自动化，无需人工标注，可轻松扩展到数十万资产。
- **实用性强**：生成的资产可直接在游戏引擎中驱动行为，展示了从生成到部署的完整流程。
- **开源友好**：项目页面、代码、数据预计公开（论文中提及项目页面https://cubepart.github.io/）。

## 8. 不足与局限

- **不支持可变形部件**：模型专注于刚性部件分解，无法处理有机角色的蒙皮权重或骨骼绑定。
- **几何穿透**：尽管交叉注意力减少了重叠，但在接触边界处仍可能出现部件相交（如图11所示），影响物理模拟。
- **空间参考歧义**：VLM标注的“左/右”“前/后”等空间形容词可能存在镜像错误或视角混淆，导致生成模型偶尔混淆对称部件或移错位置。
- **过度分割依赖源网格**：数据管道只能在现有部件边界内聚类，无法细分已融合的部件（如一体化的躯干和四肢），限制了部件粒度。
- **评测局限**：仅在200个网格的小规模测试集上评估，缺乏对大规模开放词汇泛化性的系统测试；未进行用户调查来评估部件名称的语义匹配度。
- **计算成本**：端到端推理（两阶段）约5-7秒，虽快于部分基线，但对实时应用仍显昂贵。
