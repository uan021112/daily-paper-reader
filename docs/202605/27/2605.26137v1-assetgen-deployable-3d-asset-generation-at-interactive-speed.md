## 摘要
虽然3D生成技术进展迅速，但近期的研究往往侧重于获取高分辨率资产，而将用户体验和可部署性视为次要问题。我们提出AssetGen，一个专注于这两个方面的3D生成器。给定一张参考图像，它在30秒内生成一个高质量的网格，包含烘焙法线、颜色纹理以及适合实时渲染（包括移动端用例）的受控多边形预算。AssetGen Flash变体进一步将延迟降低至14秒，适用于交互式和智能体的创建循环。我们的模型通过粗到精的VecSet框架生成物体几何，并在GPU上实现网格简化、清理和法线烘焙，以及快速的并行UV展开。然后以多视图方式生成纹理，接着进行反投影和3D修补。模型蒸馏、内核优化和流水线并行化被协同设计以加速端到端系统。我们引入了大量自动化与盲人人工评估，展示在30秒内与主流商业解决方案相比具有竞争力的视觉质量，并在不到15秒内提供预览质量的结果。最终成果是一个支持AI辅助、可在交互式工作流中部署的3D内容创建系统。

## Abstract
While 3D generation is progressing rapidly, recent work has often focused on obtaining high-resolution assets, leaving user experience and deployability as afterthoughts. We present AssetGen, a 3D generator that focuses instead on these two aspects. Given one reference image, in 30 seconds it produces a high-quality mesh with baked normals, a color texture, and a controlled polygon budget suitable for real-time rendering, including mobile use cases. The AssetGen Flash variant further reduces latency to 14 seconds for interactive and agentic creation loops. Our model generates the object geometry with a coarse-to-refine VecSet framework, which implements mesh simplification, cleaning, and normal baking on the GPU, and a fast parallel UV unwrapping. It then generates textures in a multi-view fashion, followed by backprojection and 3D inpainting. Model distillation, kernel optimization, and pipeline parallelization are co-designed to accelerate the system end-to-end. We introduce numerous automated and blind human evaluations and demonstrate competitive visual quality against leading commercial solutions in 30 seconds and preview-quality results in less than 15 seconds. The final result is a system that supports AI-assisted, deployable 3D content creation in interactive workflows.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- 采用 **显式几何优先** 的三阶段流水线：几何生成（MeshGen）→ 几何后处理（简化、隐面移除、UV展开、法线烘焙）→ 纹理生成（TextureGen）。
- 通过 **模型蒸馏**（渐进式蒸馏 + CFG蒸馏）、**内核优化**（FlashAttention-3、图编译、混合精度）、**流水线并行化**（重叠计算与I/O）等多层次优化，实现端到端的低延迟。

### 关键技术细节

1. **MeshGen（几何生成）**
   - 使用 **VecSet** 表示（一组有序潜在token），基于扩散Transformer（DiT）生成。
   - 两阶段：**粗阶段**（N=4096 tokens，24层Transformer，2.3B参数）→ **精炼阶段**（N=16384 tokens，基于粗网格采样表面点作为RoPE锚点，使用更高分辨率图像条件）。
   - 蒸馏：粗阶段从100步蒸馏到25步，精炼阶段从20步到5步；进一步通过CFG蒸馏合并无条件分支，最终只需单次前向推理。

2. **几何后处理（GPU加速）**
   - **网格简化**：基于CuMesh并行边折叠（QEM），优化了顶点位置计算（使用QEM最优位置并投影到边上），比原始中点法好，接近Houdini质量但更快（~0.46秒 vs Houdini 14秒）。
   - **隐面移除**：512视角光线投射+BVH+拓扑感知洪水填充，平均0.179秒/网格。
   - **UV展开**：提出 **并行分区UV unwrapping**（空间二分 + 重叠区域 + 合并），比串行xatlas快1.9倍，人眼几乎无区别。
   - **法线烘焙**：GPU驻留流水线（DRTK栅格化 + BVH投影），平均271ms（比Blender快10.8倍），质量相当。

3. **TextureGen（纹理生成）**
   - 多视图扩散Transformer，同时生成10个正交视图（8侧+顶+底），条件包括：参考图像（固定槽位）、每视图的法线图和位置图、文本描述。
   - 结构化多视图注意力（稀疏模式：每视图仅关注参考和邻近视图），降低计算量。
   - 蒸馏：32步 → 16 → 8 → 4步；AssetGen Flash进一步做CFG蒸馏（4步单次前向）。
   - 后处理：UV反投影（各向异性滤波）、基于可见性与入射角加权融合、3D几何引导的UV空间修补。

4. **延迟优化体系**
   - 内核级：FlashAttention-3、图编译、FP8/INT8量化（关键层保精）。
   - 流水线级：预处理（UV采样、KD树等）与扩散推理并行；MeshGen和TextureGen作为独立服务，通过共享存储传递数据。

## 7. 优点

- **系统级视角**：不是孤立优化单个模型，而是协同设计几何、纹理、后处理整个流水线，实现全局低延迟。

## 8. 不足与局限

- **单视图歧义性**：生成受限于单张输入的模糊性，对罕见类别、非对称物体、背面复杂性鲁棒性不足；对称物体较可靠。
