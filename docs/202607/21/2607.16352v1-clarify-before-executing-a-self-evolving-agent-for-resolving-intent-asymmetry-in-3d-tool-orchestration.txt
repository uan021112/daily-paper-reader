## Abstract 

A fundamental intent asymmetry plagues modern 3D asset creation: while state-of-the-art 3D toolchains demand precise, executable parameters, ordinary users typically provide vague, underspeci-fied instructions. Current 3D agents treat this ambiguity as noise, defaulting to blind execution under a single-turn assumption. To address this limitation, we introduce Clare , a clarification-aware and evolutionary 3D agent that treats intent asymmetry not as an execution error, but as an opportunity for strategic dialogue. By decoupling the generation pipeline into four specialized cognitive roles, Clare intercepts and resolves underspecified instructions before invoking computationally expensive 3D tools to seamlessly execute tasks across five diverse domains: text-to-3D generation, single-view reconstruction, multi-view reconstruction, point cloud editing, and post-processing. Crucially, rather than relying on rigid manual rules, Clare self-evolves its clarification policy via simu-lated multi-turn interactions. By optimizing a Multi-turn Reward, the agent internalizes the delicate balance between interaction effi-ciency and task completion. To rigorously test this, we construct 3D-Clarify, a comprehensive benchmark comprising 620 interac-tion scenarios with systematically injected ambiguity, missing in-formation, and mistaken details. Clare achieves state-of-the-art performance, with 60.40% and 43.34% success rates on single-step and multi-step tasks, respectively, more than doubling existing base-lines. Both quantitative and qualitative results demonstrate that proactive clarification is the missing key to robust 3D execution. ## 1 Introduction 

The democratization of 3D content creation has become increas-ingly critical across gaming, virtual reality, architectural visual-ization, and industrial design. Recent advances in 3D foundation models have enabled non-expert users to generate , reconstruct , and 

edit 3D assets through natural language interfaces [ 12 , 15 , 35 ]. How-ever, a fundamental intent asymmetry persists between what users naturally express and what 3D systems strictly require . While state-of-the-art 3D pipelines demand precise, executable parameters—such as explicit input file paths, strict output formats, appropriate model choices, and numerical quality constraints—ordinary users typically provide vague, underspecified instructions. A user might request "reconstruct this object" without providing the image, ask to "edit the chair" from a point cloud without specifying which points to keep, or inadvertently request incompatible parameters that lead to runtime failures. Current 3D agent systems operate under a single-turn, well-specified assumption : they expect complete and executable instruc-tions in one shot. When faced with underspecified requirements, these systems exhibit three failure modes: (1) blind execution of ambiguous requests, producing irrelevant outputs that waste com-putational resources; (2) undetected parameter contradictions ,causing cryptic runtime errors that frustrate users; and (3) passive behavior , lacking mechanisms to proactively identify and resolve information gaps. While general-purpose agentic frameworks like ReAct [ 33 ] and Reflexion [ 20 ] enable iterative reasoning in sym-bolic domains, they lack domain-specific mechanisms for detecting 3D-specific uncertainties—such as missing file paths, modality mis-matches, or model capability violations. Our key insight is that effective 3D task execution requires a 

progressive alignment paradigm: clarify before executing . Rather than treating ambiguity as noise to be ignored, we posit that an in-telligent 3D agent should (1) detect incomplete or mistaken require-ments before invocation, (2) engage in targeted multi-turn clarifica-tion to resolve uncertainties, and (3) leverage domain knowledge to map user preferences (e.g., "high quality", "fast") to appropriate 3D models and parameters. Crucially, the decision of when to clarify 

versus when to execute is itself learnable—excessive clarification increases interaction cost, while premature execution risks failure. To realize this vision, we introduce Clare (Clar ification-aware and Evolutionary 3D agent), a multi-agent framework (Figure 1) that transforms underspecified user requirements into executable 3D workflows through structured collaboration. Clare decouples the generation pipeline into four specialized roles: (1) Proactive Ap-prover , which gates execution by detecting ambiguity, missing infor-mation, and mistaken details; (2) Contextual Summarizer , which dis-tills multi-turn dialogue into structured task specifications; (3) 3D-Aware Programmer , which synthesizes executable code by mapping user constraints to appropriate 3D tools; and (4) Feedback Provider ,which closes the interaction loop with human-readable status re-ports and targeted clarification questions. Beyond inference-time collaboration, Clare employs a self-evolution mechanism : it learns optimal clarification policies through simulated interactions with a user agent that embodies realistic information asymmetry, opti-mizing a Multi-turn Reward (MR) that balances task completion against interaction efficiency. This simulation-based optimization enables the agent to internalize long-horizon decision making with-out manual annotation. To rigorously evaluate Clare , we construct 3D-Clarify , a com-prehensive benchmark spanning five canonical 3D domains. Each task is systematically perturbed with three types of underspecified requirements— ambiguity, missing information, and mistaken details —creating a "lazy user" simulation that tests an agent’s clar-ification capability. Our contributions are summarized as follows: 

• We propose Clare , the first multi-agent framework that systematically addresses underspecified user requirements in 3D task execution through proactive clarification and tool orchestration across five diverse domains: text-to-3D generation, single-view reconstruction, multi-view recon-struction, point cloud editing, and post-processing. 

• We introduce 3D-Clarify, a benchmark with 620 tasks and three types of controlled underspecification. By standardiz-ing the evaluation of intent asymmetry, it provides a founda-tional testbed to drive future research towards more robust, interactive 3D generation systems. 

• We develop a self-evolution training paradigm that opti-mizes clarification policies via simulated user interactions and Multi-turn Reward (MR) optimization, enabling the agent to learn effective multi-turn decision making with-out manual annotation. This empowers the agent to dra-matically elevate its execution success rate to 60.40% on single-step tasks and 43.34% on complex multi-step tasks, achieving robust performance with negligible interaction overhead. 

## 2 Related Work 2.1 LLM-based Agents for 3D Tasks 

Recent advances in large language models have enabled language-driven control over complex software systems, including 3D gen-eration and reconstruction pipelines. In parallel, numerous agent reasoning frameworks have been employed to enable LLMs to interact with the external environment. For example, ReAct [ 33 ]interleaves reasoning traces with task-specific actions, allowing the agent to dynamically interact with the environment. Reflexion [ 20 ]further introduces a feedback-driven learning mechanism, enabling agents to improve subsequent decisions by reflecting on execution outcomes and past failures. Existing LLM-based 3D agents [ 3, 6 , 21 , 31 ] are mostly designed around specific 3D workflows rather than diverse task settings. By comparison, Clare targets a substantially broader task space, seam-lessly integrating text-to-3D, reconstruction, point cloud editing, and post-processing. This capability to handle multi-step composi-tions makes our framework not only clarification-aware but also highly generalizable across unified 3D executions. ## 2.2 Interactive Clarification 

Interactive clarification has been widely studied in human-centered AI systems [ 1 , 2, 17 , 18 , 22 ], where systems must actively gather missing information through multi-turn interactions before execut-ing a task. Existing work on interactive clarification can be broadly cate-gorized into three directions. First, methods focusing on informa-tion elicitation ask follow-up questions to resolve underspecified constraints [ 36 ]. Second, user-centric frameworks explore interac-tion policy learning to balance task completion against dialogue cost [ 7, 14 , 18 ]. Third, collaborative paradigms shift from passive responses to active goal-oriented cooperation [ 10 , 37 ]. Collectively, these efforts establish clarification as a core decision-making capa-bility. Our work differs by situating clarification specifically within the fragile pipeline of 3D task execution. We treat dialogue not as an auxiliary behavior, but as a prerequisite to prevent invalid tool selections, parameter contradictions, and fatal downstream errors. 

## 2.3 Self-Evolving Agents 

Recent surveys [ 4 , 9] suggest that the development of self-evolving agents reflects a paradigm shift from static systems to adaptive, lifelong agents that continuously improve through interaction. However, self-evolving frameworks [ 34 , 41 ] primarily focus on general language-centric reasoning, leaving domain-specific work-flows like 3D asset creation underexplored. Our work fills this gap by introducing a 3D-domain-aware self-evolution paradigm. By coupling the agent with a stateful simulated user, we optimize clar-ification policies through multi-turn trajectories. This allows the system to internalize the delicate balance between interaction effi-ciency and reliable execution under underspecified requirements.

## 4 The CLARE Framework: Coordination via Progressive Alignment 4.1 Architecture Overview 

Handling 3D tasks involves inherent ambiguity: users often lack the precise terminology to describe spatial geometry or rendering styles. We introduce Clare , a multi-agent framework designed not merely to execute commands, but to manage the uncertainty of human intent. Figure 2 shows a progressive alignment loop that decouples the complex generation process into four specialized cognitive roles: Decision Gating , Context Refinement , Executable Synthesis , and Feedback Alignment . This separation of concerns ensures that ambiguous or mistaken requirements are trapped and resolved before touching the computationally expensive 3D tools. 

## 4.2 Core Components 

Proactive Approver: The Cognitive Gatekeeper. Traditional agents often hallucinate parameters for vague instructions, leading to wasted generation time. The Proactive Approver acts as the first line of defense. It evaluates the completeness of the current instruction 𝑞 𝑗 

against the conversation history. Instead of blindly forwarding re-quests, it computes a binary decision state 𝑑 ∈ { Execute , Clarify }.By detecting three distinct types of requirements— Ambiguity , Miss-ing Information , and Mistaken Details —the Approver effectively "freezes" the execution pipeline until the user’s intent is fully ac-tionable, enforcing a rigorous standard for task initiation. 

Contextual Summarizer: Structured Intent Mining. Once approved, the Contextual Summarizer bridges the gap between fragmented natural language and structured execution specifications. It func-tions as the system’s dynamic memory, aggregating the multi-turn dialogue history H< 𝑗 into a normalized task manifest M. This com-ponent performs crucial state tracking : it resolves co-references (e.g., "remove it " referring to a previously generated chair) and explicitly extracts constraints such as target file paths and quality preferences. By distilling noise into a structured schema, it provides a noise-free specification for the downstream programmer.

3D-Aware Programmer goes beyond generic code generation by incorporating a model selection knowledge base. It understands the capability boundaries of different 3D tools. ## 4.3 Self-Evolution via Collaborative Simulation 

Drawing inspiration from collaborative training frameworks [ 28 ], we treat the 3D task execution as a long-term optimization problem. Static datasets often fail to capture the nuances of when to clarify versus when to execute. We address this by evolving the agent’s policy 𝜋 through a Collaborative Simulation Loop , maximizing a multi-turn reward derived from interactive trajectories with a dynamic user simulator. 

4.3.1 Interactive Forward Sampling. To estimate the long-term util-ity of a clarification strategy, Clare performs lookahead via simula-tion. Given a context 𝑥 = (𝑞 𝑗 , H<𝑗 ), the agent samples 𝑁 candidate responses {𝑚 (1) , . . . , 𝑚 (𝑁 ) } from its current policy 𝜋 𝜃 . For each can-didate, we roll out a full interaction trajectory 𝜏 (𝑖 ) by coupling the agent with a Simulated User U. This process reveals downstream consequences, helping the agent separate vital clarifications from redundant, costly queries. 

Simulated User with Hidden Goals. The Simulated User U is not a static responder but a stateful agent designed to mimic realis-tic human behavior. Initialized with a hidden target goal G∗ (e.g., specific quality and format constraints) and a surface-level vague query 𝑞 0, U operates under an Information Asymmetry setting. As detailed in our implementation, it follows strict behavioral stages: initially providing vague requirements and only revealing critical parameters when the agent explicitly identifies the information gap. This "interaction friction" forces the agent to learn precise interrogation strategies to unlock the hidden goal.

## 5.2 Phase 2: Sequential Multi-Task Composition 

Real-world 3D workflows often involve sequential dependencies. To simulate this, we construct 120 multi-step samples by chain-ing atomic tasks based on 15 distinct workflow templates (e.g., 

Reconstruction → Editing → Outlier Removal ). To mirror the nat-ural ambiguity of human communication in sequential pipelines, we intentionally obfuscate the explicit file paths for intermediate inputs. Rather than providing absolute paths for downstream op-erations, the simulated user employs contextual co-references to denote intermediate results. This design explicitly challenges the agent to maintain a persistent state, requiring it to accurately re-solve cross-turn dependencies and correctly map previous outputs to subsequent inputs to successfully execute the entire workflow.

## 6 Experiments 6.1 Experiment Setup 

To rigorously evaluate the capability of agents in handling realistic user requirements, we utilize our proposed 3D-Clarify bench-mark, which comprises 500 single-step and 120 multi-step tasks, each associated with three underspecified queries. Diverging from single-turn paradigms, we emphasize dynamic goal fulfillment by leveraging Qwen3-8B as a stateful user simulator to emulate au-thentic, multi-turn behaviors conditioned on the initial query, final objective, and dialogue history. Our agent components are similarly powered by Qwen3-8B. To enable adaptive tool orchestration, we construct a compre-hensive model knowledge base by analyzing a diverse suite of integrated 3D toolsets, including Worldgen [ 30 ], Trellis [ 29 ], Tri-planeturbo [ 16 ], Hunyuan-3D [ 23 –25 ], VGGT [ 26 ], Fast3R [ 32 ], Con-certo [ 38 ], and Open3D [ 40 ]. For self-evolution, we perform LoRA-based fine-tuning on the Proactive Approver using a de-duplicated subset of 3,000 multi-turn trajectories filtered by the Multi-turn Reward. Both Supervised Fine-Tuning (SFT) and Direct Preference Optimization (DPO) are conducted using the LLaMA-Factory [ 39 ]framework on two NVIDIA RTX 3090 GPUs for 2 epochs, utilizing a learning rate of 5 × 10 −6 and a total batch size of 4. For a fair com-parison, all baselines employ the identical Qwen3-8B backbone. For 

Clare and the original baselines, inference uses temperature 0.6

and top_p 0.9; the structured-output adapters use temperature 0.2

and top_p 0.9, as detailed in Appendix A.4. All interactive methods use a maximum interaction limit of 𝑇 𝑚𝑎𝑥 = 8 turns. In the reward formulation, the penalty coefficient 𝜆 is set to 0.1.Detailed agent prompts, tool specifications, benchmark construc-tion, and model-selection knowledge are provided in Appendices A and B. 

## 6.2 Baselines 

To evaluate the effectiveness of our proposed framework, we com-pare against static single-turn prompting methods (Zero-shot, Few-shot, and Chain-of-Thought [ 27 ]) and dynamic, feedback-driven agentic frameworks (ReAct [ 33 ] and Reflexion [ 20 ]). We further include three task-interface-matched related-work adapters: CLAM-BER [ 36 ], which uses a single clarification-aware agent; CEP [ 37 ], which follows an ask-before-plan state collection, clarification, and planning flow; and 3D-GPT [ 21 ], which follows task dispatch, con-ceptualization, and modeling. These adapters retain their defining control flows while using the same Qwen3-8B backbone, simulated user, 3D tool interface, turn budget, and evaluator as Clare ; they are adaptations to our executor rather than exact reproductions. Full implementation details are provided in Appendix C.1.

## 6.4 Quantitative Results 

Table 2 presents the comparison on the 3D-Clarify benchmark. We emphasize Task Success Rate (SR) as the primary metric for flawless execution, while Goal Completion Rate (CR) reflects partial parametric accuracy. Our analysis yields three key findings. 

Proactive Clarification is Essential for Underspecified Requirements. 

For single-step tasks, single-turn paradigms (Zero-shot, Few-shot, CoT) nearly collapse on ambiguity and missing-info tasks, achiev-ing only 0–6% SR despite moderate CR scores (31–35% and 17–18%, respectively). Dynamic agentic frameworks (ReAct, Reflexion) show similar failures (1.6–8.4% SR on ambiguity/missing tasks). Adding clarification-aware or 3D-specific scaffolds improves some cases, but the best adapter reaches only 24.40% average SR on single-step tasks and 9.44% on multi-step tasks. In contrast, Clare-base 

achieves 48.60% and 27.22% average SR, respectively. This gap indi-cates that an ask-first prompt or standard 3D decomposition alone is insufficient; reliable execution also requires explicit pre-execution gating and persistent clarified state. 

Self-Evolution via MR Optimization Substantially Improves Clari-fication Policies. Multi-turn reward optimization yields significant gains in both SR and CR. On ambiguity single-step tasks, Clare-SE-DPO achieves 60.00% SR (CR: 70.52%) compared to Clare-base ’s 45.80% SR—a 14.20% absolute improvement. On missing-info tasks, 

Clare-SE-SFT reaches 52.00% SR (CR: 69.47%), while Clare-SE-DPO achieves 59.40% SR (CR: 75.32%). For mistaken tasks, Clare-SE-SFT attains the highest 66.80% SR (CR: 85.43%). Averaged across all single-step underspecifications, Clare-SE-DPO achieves 60.40% SR (CR: 73.51%), more than doubling the strongest baseline (CEP: 24.40% SR) and demonstrating that clarification policies are learn-able through simulated interaction. 

Multi-Step Tasks Amplify the Advantage of Structured Clarifica-tion. The transition to multi-step workflows reveals a critical find-ing: baseline methods collapse to 0% SR on ambiguity and missing-info multi-step tasks, while Clare-SE-DPO maintains 34.17–49.17% SR (CR: 53.41–68.74%). This 34–49% absolute gap underscores that multi-step workflows require consistent clarification across turns—a capability enabled by Clare ’s Contextual Summarizer for maintain-ing structured task state. On mistaken multi-step tasks, Clare-SE-SFT achieves 48.33% SR (CR: 83.77%), demonstrating that early-stage clarification prevents error propagation. Overall, Clare-SE-DPO 

attains 43.34% average SR (CR: 64.97%) on multi-step tasks, com-pared to 4.44–7.78% SR for dynamic baselines and 11.39–13.89% SR for single-turn paradigms. While absolute performance indicates room for growth (e.g., larger backbones), Clare represents the first demonstration that proactive clarification is the key to reliable 3D task execution under underspecified requirements. 

Performance Analysis on Mistaken Details. We observe a striking divergence between Goal Completion Rate (CR) and Task Success Rate (SR) across baselines on mistaken tasks. For instance, single-step Zero-shot achieves 77.55% CR but only 34.80% SR. Because the underlying LLM possesses strong instruction-following capa-bilities, baselines successfully configure most standard parameters, artificially inflating CR. However, lacking proactive clarification, they blindly execute intentionally injected fatal errors, causing the overall execution to collapse. Conversely, our framework intercepts these errors, though CLARE-SE-SFT outperforms CLARE-SE-DPO (66.80% vs. 61.80% single-step SR). We attribute this to the inher-ent conflict between preference alignment and error correction. SFT directly clones robust "firm correction" trajectories via max-imum likelihood estimation. In contrast, preference optimization (DPO) biases the model toward compliant, agreeable responses. Consequently, when confronted with explicit user errors, the DPO agent experiences policy degradation in the challenging action of proactively contradicting the user, leading to lower success rates.

## 6.8 Effect of Training Data Scale 

We investigate the impact of training data scale on self-evolution performance across 120 multi-step tasks. This divergence suggests that preference-based optimization excels at refining the decision boundary for task success even after parametric learn-ing saturates. ## 7 Conclusion 

We present Clare , a self-evolving, clarify-before-execute multi-agent framework that resolves underspecified requirements in 3D tool orchestration, together with the 620-task, five-domain 3D-Clarify benchmark. Across static prompting, dynamic agentic meth-ods, and task-interface-matched CLAMBER, CEP, and 3D-GPT adapters, Clare consistently improves execution reliability by clar-ifying ambiguity, missing information, and mistaken constraints Clarify Before Executing: A Self-Evolving Agent for Resolving Intent Asymmetry in 3D Tool Orchestration 

before tool use. Its MR-optimized policy reaches 60.40% SR on 500 single-step tasks and 43.34% SR on 120 multi-step tasks, more than doubling the strongest baseline on single-step tasks. These results show that proactive clarification policies can be learned and opti-mized through simulated multi-turn interaction, offering a practical path toward more reliable human–AI collaboration in specification-heavy 3D workflows. 

Limitations. The overall execution success of Clare is inher-ently bounded by the capabilities of the backbone LLM and the integrated 3D toolchains. Additionally, while our simulated environ-ment effectively optimizes clarification policies, fully capturing the diversity and unpredictability of real-world human queries remains an ongoing challenge.

## A Method Details A.1 Architecture 

To effectively manage the uncertainty of human intent, CLARE employs a multi-agent progressive alignment loop that decouples the 3D generation pipeline into four specialized cognitive roles. This collaborative architecture ensures that requirements are fully specified before invoking computationally expensive 3D tools. 

• Proactive Approver (Cognitive Gatekeeper): Acting as the first line of defense, the Approver evaluates the complete-ness of the user’s current instruction against the multi-turn conversation history. Instead of defaulting to blind execu-tion, it computes a binary decision state to detect ambiguity, missing information, or mistaken details. The Approver out-puts its decision in a strict XML format (<judgement> and <reason>), effectively freezing the execution pipeline and categorizing the rejection (e.g., unclear objectives, invalid file paths) until the user’s intent becomes fully actionable. 

• Contextual Summarizer (Structured Intent Mining): Once a request is approved, the Summarizer acts as the sys-tem’s dynamic memory by aggregating fragmented multi-turn dialogue into a normalized task manifest. It performs essential state tracking and co-reference resolution. The output is strictly formatted into XML fields (<task_type>, <task_summary>, <file_path>, and <extra_info>), which distills noisy conversational context into explicit execution constraints and provides a noise-free specification for the downstream code synthesis. 

• 3D-Aware Programmer (Adaptive Tool Orchestration): The Programmer is responsible for translating the structured manifest into executable Python code. Unlike generic code generators, it incorporates a built-in Model Selection Knowl-edge Base to understand the capability boundaries and in-put/output formats of different 3D tools. It dynamically maps user-defined constraints (such as speed and quality) to specific models T3𝐷 and execution parameters 𝜃 𝑒𝑥𝑒𝑐 . This ensures that the synthesized code is not only syntactically correct but also physically and semantically grounded in the 3D generation context. 

• Interaction Feedback Provider (Closing the Semantic Gap): This module serves as the interface for conversational align-ment, translating internal system states into human-readable, actionable feedback. It operates in two adaptive modes. In Clarification Mode, it intercepts the Approver’s rejec-tion reasoning and translates it into targeted follow-up questions, guiding the user to provide missing parameters without cognitive overload. In Status Reporting Mode, it interprets raw execution logs and potential runtime errors post-execution, providing the user with concise summaries and clear next-step suggestions to iteratively refine their 3D assets. 

## A.2 Details and Prompts of CLARE Agents 

A.2.1 Proactive Approver. The Approver is the core role responsi-ble for "decision gating." In each round of dialogue, it receives the current user input and the existing dialogue history, combines them into a complete requirement, and then determines whether the re-quirement meets the "executable" condition. "Executable" means that the requirement must contain a clear task objective (four types of 3D workflows), corresponding input information (such as image file paths or text prompts), and necessary parameters and output format. In addition, for generation/reconstruction classes, a clear output format is required (such as .ply/.glb), for post-processing classes, specific operations are required (such as removing out-liers/converting meshes to point clouds), and for point cloud edit-ing classes, the "detail objects" to be edited are required (such as "remove chair/keep table"). When the task is sufficiently specific, Approver must output strict XML: ‘<judgement> Yes</judgement>‘. If the task still contains missing, ambiguous, or erroneous elements, it must output ‘<judgement> No</judgement>‘ along with a ‘<rea-son>‘ in the same output, where the reason is selected from pre-defined rejection categories (Missing information / Task objective unclear / Mistaken details / User wants to clarify or confirm the task details). Furthermore, Approver’s output is protocolized to a format that "must contain only these XML tags," ensuring stable and reliable routing. Assistants, as the core of the Agent framework, support the use of trained preference models (e.g., via DPO/SFT), allowing them to directly influence the quality of gating decisions.

A.2.3 3D-Aware Programmer. After receiving the structured task summary, the Programmer first parses the task type and retrieves candidate model information matching the current task type from the built-in Model Selection Knowledge Base. It then injects the output format and capability features of these models as priors for model selection into the prompts, constraining the LLM to follow correct tool selection and parameter constraints when generating code. Subsequently, it generates a directly executable Python code call sequence according to preset template rules, and performs post-processing with regularization to correct key call forms, thereby improving the executability of the generated code and its consis-tency with downstream 3D tool interfaces.

Supervised Fine-Tuning (SFT) Setup. For the SFT phase, we fine-tune the agent on a curated dataset of approximately 3,000 high-quality trajectories (filtered by Multi-turn Reward). The objective here is to clone the behavior of the top-ranked clarification trajectories via standard maximum likelihood estimation. 

Direct Preference Optimization (DPO) Setup. For the DPO phase, the dataset consists of approximately 3,000 pairwise comparisons constructed from the simulated rollouts, where "chosen" and "re-jected" trajectories are distinguished based on their composite Multi-turn Reward. We employ the standard Sigmoid preference loss. The KL divergence penalty coefficient is set to 𝛽 = 0.1 to prevent the model from deviating too far from the reference policy. ## B.3 Data Curation and Source Assets 

To support a rigorous and comprehensive evaluation of 3D agent capabilities under both atomic and compositional settings, we con-struct a curated benchmark consisting of two subsets: a single-step set with 500 samples and a multi-step set with 120 samples. Both subsets are designed to uniformly cover five canonical 3D task do-mains: text-to-3D generation , single-view reconstruction , multi-view reconstruction , point cloud editing , and post-processing . This design follows a domain-balanced construction principle to ensure that agent performance is evaluated across diverse yet representative 3D workflows. 

Single-Step Task Construction. We first construct a set of 500 atomic tasks, where each task corresponds to a single executable operation without intermediate dependency. To ensure balanced coverage, we allocate 100 samples to each of the five task domains. Each sample is instantiated as a structured triplet consisting of input specification, target goal, and corresponding asset reference, enabling deterministic evaluation of execution correctness. 

Multi-Step Task Construction. To simulate realistic 3D workflows that involve sequential dependencies, we further construct 120 multi-step samples by composing multiple atomic tasks. These compositions span cross-domain pipelines (e.g., reconstruction →

editing → post-processing), requiring the agent to maintain consis-tent state and intermediate representations across steps. Compared to single-step tasks, these samples emphasize long-horizon reason-ing and inter-task coordination. 

Text-to-3D Generation. For the text-to-3D generation domain, we adopt data from the TriplaneTurbo [ 16 ] dataset. This dataset provides high-quality text prompts. We select samples with diverse semantic categories and varying levels of geometric complexity to ensure broad coverage. 

Reconstruction Tasks. We consider both single-view and multi-view reconstruction under two distinct settings: object-level and scene-level reconstruction. 

• Object-level reconstruction is based on the DTU [ 11 ] dataset, which provides calibrated multi-view images and ground-truth geometry for precise object reconstruction. 

• Scene-level reconstruction utilizes data from Mip-NeRF 360 [ 5]and Tanks and Temples [ 13 ], both of which contain large-scale real-world scenes with complex geometry and view-point distributions. 

Point Cloud Editing. For point cloud editing tasks, we source data from the ScanNet [ 8] dataset, enabling evaluation of editing operations such as keeping, segmentation, and deletion. 

Post-Processing. The post-processing tasks are constructed us-ing a combination of real-world and synthetic assets. Specifically, we utilize ScanNet [ 8 ] data together with internally created assets in multiple formats. This diverse set of representations enables systematic evaluation of downstream operations, such as format conversion and outlier removal across heterogeneous 3D data for-mats. 

## B.4 Injection Details 

The core contribution of 3D-Clarify is the controlled injection of user uncertainty. We apply three distinct perturbation strategies to the ground-truth goals, creating a "Lazy User" persona that tests the agent’s ability. 

Ambiguity Injection (Semantic & Referential Vagueness): We sim-ulate user uncertainty via a obfuscation strategy using task-specific templates. We apply semantic bleaching by replacing domain-specific verbs with generic actions and specific targets with abstract nouns, while stripping all parametric constraints to force context-based inference. 

Missing Information (Value Masking): We simulate user negli-gence through a template-based value masking strategy. We replace concrete execution parameters with indefinite placeholders while retaining the structural intent. This compels the agent to detect that while the necessary parameter keys are mentioned, their executable values are absent, necessitating targeted clarification. 

Mistaken Details (Constraint Violations): Unlike simple missing info, these samples simulate erroneous user beliefs that require cor-rection rather than just filling. We synthesize three sub-categories of mistakes: (1) Contextual Contradiction: We inject conflicting attributes where the user demands properties that are mutually ex-clusive for the underlying model. (2) Input Modality Mismatch: We alter input file extensions to simulate modality errors, such as providing a 2D image (.jpg ) for a Point Cloud Editing task (which strictly re-quires 3D .ply input), forcing the agent to check file type compatibility. (3) Invalid Parameter Values: We corrupt numerical fields with out-of-bound or nonsensical values. This tests the agent’s ability to validate arguments against API constraints before execution.

C.1.3 CoT. To evaluate whether explicit reasoning mitigates the effects of underspecified requirements, we implement a Chain-of-Thought (CoT) baseline. We modify the Few-Shot prompt to man-date a structured reasoning phase prior to code synthesis. Specifi-cally, the model is instructed to generate a <plan> that systemati-cally analyzes the task type, input parameters, output paths, and explicitly justifies its model selection by mapping user constraints (e.g., quality and speed) to the tool’s feature set. While this step-by-step deduction improves internal logical consistency and helps the agent accurately select tools when requirements are fully specified, it fundamentally remains a single-turn paradigm. When critical information is absent or ambiguous, the CoT agent attempts to "reason" its way out of the uncertainty using the same hardcoded heuristics (such as path slugification and format overriding) rather than pausing to elicit missing facts. Consequently, it still succumbs to the fatal errors of blind execution.

C.1.4 ReAct. Transitioning from static paradigms to dynamic agen-tic frameworks, we implement the ReAct baseline to evaluate general-purpose iterative reasoning. In this setup, the language model is prompted to interleave reasoning traces (Thought) with specific task executions (Action) based on the current context (Observation). Crucially, to provide a fair comparison against our clarification-aware framework, we explicitly expand the ReAct agent’s action space to include a Clarify action alongside the standard Generate 

command. This design theoretically empowers the agent to pause the execution pipeline and interrogate the user when it detects missing information, ambiguity, or mistaken details. However, as demonstrated in our quantitative results, without a decoupled, rig-orous gating mechanism (like CLARE’s Proactive Approver) and specialized multi-turn interaction training, the general-purpose ReAct agent often struggles to determine the optimal boundary between reasoning and asking, frequently underutilizing the clari-fication action and defaulting back to premature execution.

C.1.5 Reflexion. Furthermore, Reflexion introduces an episodic memory buffer for self-reflection. When the agent’s generated code fails to execute or violates constraints, an automated ver-ifier provides specific error reasons. The agent then executes a 

self_reflect function to synthesize actionable lessons (e.g., "I should..."), which are subsequently injected into its system prompt for the next trial. While this enables the agent to learn from its mistakes over multiple attempts (up to 3 trials in our setup), it fundamentally relies on post-execution recovery rather than pre-execution prevention . As demonstrated in our evaluation, this trial-and-error approach is less efficient and less robust than CLARE’s proactive clarification mechanism, particularly when dealing with fatal modality mismatches or unrecoverable parameter errors that could have been prevented prior to tool invocation.

## C.2 Experiment Details 

C.2.1 Ablation details. We present detailed ablation results across the three underspecified requirement types (Ambiguity, Missing Info, Mistaken) for both single-step and multi-step tasks. Table 2 in the main paper summarizes averaged results, while Table 4 provides fine-grained results for each task complexity and uncertainty type. We evaluate two ablation variants. In w/o Summarizer , the Contextual Summarizer is disabled and the agent operates with raw dialogue history without structured state maintenance. In w/o Ap-prover , the Proactive Approver is completely removed, reducing the system to blind execution equivalent to ReAct. Both variants re-tain the identical Qwen3-8B backbone and inference configuration as Clare-base for fair comparison. Removing the Proactive Approver causes catastrophic perfor-mance degradation, validating that pre-execution gating is the foun-dation of clarification capability. On single-step tasks, average SR drops from 48.60% to 27.67% ( -20.93% ). Breaking down by task type, the decline is most severe on Missing Info tasks (41.00% to Zhu et al. 

16.20%, -24.80% ), confirming that placeholder detection fundamen-tally requires pre-execution validation. Ambiguity tasks show simi-lar degradation (45.80% to 22.60%, -23.20% ), as semantic vagueness goes unresolved without explicit detection. Mistaken tasks exhibit the smallest decline (59.00% to 44.20%, -14.80% ), since some contra-dictions can still be caught via runtime errors. On multi-step tasks, degradation intensifies: average SR drops from 27.22% to 13.06% (-14.16% ), with Missing Info collapsing to merely 0.83%, demon-strating that multi-turn information gaps become nearly impossible to resolve without proactive detection. The Contextual Summarizer exhibits clear complexity-dependent utility, with negligible impact on single-step tasks but substantial contribution to multi-step workflows. On single-step tasks, removal causes minimal degradation (48.60% to 48.20%, -0.40% ), confirm-ing that atomic instructions require no structured state. However, on multi-step tasks, the impact becomes pronounced: average SR drops from 27.22% to 20.28% ( -6.94% ). Ambiguity proves most sen-sitive (23.33% to 6.67%, -16.66% ), as multi-turn semantic resolution demands persistent tracking of clarified constraints. Interestingly, Missing Info shows slight improvement (22.50% to 25.83%, +3.33% ), suggesting that for certain missing-information patterns, raw dia-logue history may preserve richer contextual cues than summarized state. Mistaken tasks show moderate decline (35.83% to 28.33%, -7.50% ), indicating cross-turn contradiction detection benefits from structured representation. These findings reveal distinct task-type-specific vulnerabilities and inform targeted improvements. The Approver’s dominant con-tribution suggests prioritizing its enhancement through more so-phisticated detection mechanisms or domain-specific validators. The Summarizer’s complexity-dependent utility motivates adaptive designs: lightweight summarization for single-turn queries and full structured state for multi-turn workflows. The resilience of Mistaken tasks supports hybrid detection combining pre-execution validation with post-execution error analysis. Together, the abla-tion study validates our modular architecture where the Approver provides essential gating and the Summarizer enables scalable multi-turn coordination. 

## C.3 Statistical Analysis of Failure Modes 

Figure 8 visualizes the failure reason distribution across the three underspecified requirement types. Several patterns emerge: 

Ambiguous Tasks: Dominated by Stuck Errors. Ambiguous queries exhibit the highest stuck rate (88 occurrences, 41.7% of ambiguous failures). This aligns with the nature of semantic vagueness: when user requests lack concrete targets (e.g., “create something”), agents struggle to terminate clarification and commit to execution. The second most common failure is model selection (52 occurrences, 24.6%), suggesting that even after clarification, agents often misinfer the appropriate foundation model from underspecified semantic cues. 

Missing Info Tasks: Input Path Validation Failures. Missing in-formation queries show a distinctive spike in input_path errors (69 occurrences, 32.7%), followed by model errors (60 occurrences, 28.4%). This pattern reflects the structural nature of missing-info perturbations: when parameter values are masked with placehold-ers, agents frequently fail to resolve input file dependencies, leading to execution failures. The high model error rate indicates that path resolution and model selection are coupled challenges in under-specified scenarios. 

Mistaken Tasks: Parameter and Stuck Errors. Mistaken detail queries exhibit a bimodal distribution: parameter errors (63 occurrences, 32.6%) and stuck errors (63 occurrences, 32.6%) dominate equally. This reflects the dual challenge of handling contradictory con-straints: agents either attempt to execute with invalid parameters (e.g., incompatible file formats, out-of-bound values) or become trapped in clarification loops when attempting to resolve unresolv-able contradictions. The lower model error rate (35 occurrences, 18.1%) suggests that mistaken tasks often provide sufficient seman-tic grounding for model selection, even when other parameters are erroneous.
