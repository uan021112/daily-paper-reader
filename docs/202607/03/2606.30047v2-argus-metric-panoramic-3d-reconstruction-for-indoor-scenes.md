## 摘要
由于缺乏大规模全景RGB-D训练数据，用于全景数据的度量前馈三维重建仍未被充分探索。我们提出了Realsee3D，这是一个包含1万室内场景（1千真实场景，9千合成场景）、29.9万全景视点以及精确度量标注的混合数据集，以及Argus——一个在此数据集上训练的用于度量全景三维重建的前馈网络。在Realsee3D的稀疏无序捕获设置中，一个选择不当的坐标锚点可能导致全局位姿漂移。Argus通过一个学习到的共视模块来解决这个问题，该模块选择几何上最优的参考视图来锚定度量世界坐标系。为了进一步改进多任务学习，我们将双向像素到世界映射分解为可解释的子步骤，每个子步骤都有监督和跨坐标联合约束，增强了预测分支之间的几何一致性。在Realsee3D基准上，Argus在相机位姿估计、深度估计和点云重建方面达到了最先进的度量性能。项目页面：https://argus-paper.realsee.ai。

## Abstract
Metric feed-forward 3D reconstruction for panoramic data remains under-explored due to the lack of large-scale panoramic RGB-D training data. We present Realsee3D, a hybrid dataset of 10K indoor scenes (1K real, 9K synthetic) with 299K panoramic viewpoints and precise metric annotations, and Argus, a feed-forward network trained on it for metric panoramic 3D reconstruction. In the sparse unordered capture setting of Realsee3D, a poorly chosen coordinate anchor can cause global pose drift. Argus addresses this with a learned covisibility module that selects the geometrically optimal reference view to anchor the metric world frame. To further improve multi-task learning, we decompose the bidirectional pixel-to-world mapping into interpretable sub-steps with per-step supervision and cross-coordinate joint constraints, reinforcing geometric consistency across prediction branches. On the Realsee3D benchmark, Argus achieves state-of-the-art metric performance in camera pose estimation, depth estimation, and point cloud reconstruction. Project page: https://argus-paper.realsee.ai.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：基于 Transformer 的前馈三维重建（如 DUSt3R、VGGT）在透视图像场景取得突破，但全景图像因覆盖全视场、跨视图重叠大，更契合稀疏视图前馈范式的需求，具有高实用价值。
- **问题**：全景数据的度量（metric）前馈三维重建仍未被充分探索，主要瓶颈在于**缺乏大规模、高质量、含精确度量标注的全景 RGB-D 训练数据**。
- **含义**：现有前馈模型（如 VGGT）在透视数据上训练，直接用于全景数据性能严重下降；且稀疏无序捕获中，固定启发式参考视图导致全局位姿漂移、空间不一致重建。本文提出**Realsee3D**数据集和**Argus**网络，填补该空白，为全景度量重建建立新标杆。

## 2. 方法论：核心思想、关键技术细节

### 2.1 整体框架
- 输入：稀疏无序的多视角全景 ERP（Equirectangular）图像。
- 骨干：DINOv2 提取 patch tokens，后接**共视 Transformer**（2层交替注意）和**几何 Transformer**（24层交替注意）。
- 输出：每个视图的相机位姿（6-DoF + 置信度）、深度图、多坐标系点图（相机坐标系、旋转坐标系、世界坐标系）。

### 2.2 学习选择参考视图（Covisibility-guided Reference Selection）
- **问题**：稀疏无序输入下，固定参考视图（如第一帧）可能位于角落或边界，导致全局位姿漂移。
- **方法**：
  - 根据真实位姿和深度预计算所有视图对的共视分数，构建全局共视矩阵，用 Dijkstra 算法选择**全局连通性最优**的视图作为真实参考标签（one-hot）。
  - 共视 Transformer 预测每个视图的共视得分，训练时用二元交叉熵（BCE）监督；推理时取 argmax 选取最高分视图作为参考帧。
  - 交换所选参考视图 tokens 与固定第一帧 tokens，实现等价效果，同时保留近似置换不变性。

### 2.3 全景几何分解（Panoramic Geometric Factorization）
- **动机**：直接回归世界坐标点图将多种几何变换混合，增加优化难度，限制多任务协同。
- **分解步骤**（以单个视图为例，图 3）：
  1. **深度 → 相机点**：\(P_c = D \odot P_u\)，其中 \(P_u\) 为球面单位像素点。
  2. **相机点 → 旋转点**：\(P_r = R P_c\)（旋转到参考坐标系）。
  3. **旋转点 → 世界点**：\(P_w = P_r + t\)。
  4. 逆变换类似：\(P_r = P_w - t\)，\(P_c = R^{-1} P_r\)，\(D = \|P_c\|_2\)。
- 每个中间表示（\(D, P_c, P_r, P_w\)）由独立的 DPT 头预测，并施加独立监督 + 跨坐标联合约束（见损失函数）。

### 2.4 损失函数
- **共视损失** \(L_{covis}\)：BCE 监督参考视图选择。
- **相机损失** \(L_{cam}\)：旋转 L1 + 平移 L1，均带置信度加权和 \(\log\) 正则项（\(\alpha=0.2\)）。
- **深度损失** \(L_d\)：L2 不确定性损失（预测与真实深度 + 梯度项），置信度加权。
- **点图损失** \(L_p\)：类似深度损失，含点图 L1 + 法向约束。
  - 分别用于相机点 \(L_{cp}\)、旋转点 \(L_{rp}\)、世界点 \(L_{wp}\)。
- **几何联合损失** \(L_{joint}\)：利用预测的 \(D, P_c, P_r, P_w, R, t\)，沿前向/逆向变换链计算相邻步骤的监督（共6项），强制跨分支几何一致性。
- **总损失**：\(L = 0.1 L_{covis} + 5.0 L_{cam} + L_d + L_{cp} + L_{rp} + L_{wp} + L_{joint}\)。

### 2.5 预测与数据预处理
- 深度头用 \(\exp\) 激活保证正值；点图头用 inverse log transform 处理宽动态范围。
- ERP 图像缩放到 560×280，裁剪上下 15% 得到 560×196，并做颜色抖动、高斯模糊、灰度化及绕 Y 轴随机旋转（相当于水平像素平移）。

## 3. 实验设计

### 3.1 数据集与基准
- **Realsee3D 数据集**：
  - 真实子集：1000 个真实室内场景，9483 个房间，24263 个全景视点（LiDAR 采集 + 手动校正）。
  - 合成子集：9000 个程序生成场景（基于真实房型分布），86479 个房间，274810 个视点（Unreal Engine 5 光线追踪渲染，五种光照方案）。
  - 场景级 9:1 划分为训练/测试。
- **基准任务**：相机位姿估计、多视图深度估计、点云重建（均支持度量评估）。

### 3.2 对比方法
- 因缺少现成全景前馈模型，将 SOTA 透视模型微调到 Realsee3D：**VGGT360**（VGGT）、**MapAnything360**（MapAnything v1.1.1）、**π3\_360**（π3）。
- 所有对比模型使用相同的训练/测试划分、预处理和训练迭代数。
- 传统 SfM 方法（COLMAP、OpenMVG）因速度慢且结果不可用，被省略。

### 3.3 评估指标
- 位姿：AUC、RRA、RTA（5°/10°/20°）、ATE RMSE、接受率（旋转<10°且平移<0.5m）。
- 深度：AbsRel、RMSE、MAE、δ1（<1.03）、δ2（<1.25），分别在 IRLS 对齐、中值对齐、无对齐（度量）三种方案下报告。
- 点云：Accuracy、Completeness、Normal Consistency（ICP 对齐后或度量评估）。

## 6. 主要结论与发现

- Argus 在相机位姿估计（ATE 降低 28%–69% vs. MapAnything360）、深度估计（AbsRel 0.019 合成，0.048 真实）和点云重建（度量评估下显著优于所有基线）上均达到最优度量性能。
- 参考视图学习有效抑制全局位姿漂移（ATE 从 0.060 降至 0.027）。
- 几何分解与跨坐标联合监督显著提升多任务协同，尤其相机坐标点图损失对深度学习贡献最大。
- 虽然在结构化室内场景表现优异，但在无界室外/航拍场景泛化能力受限，且大视点集（数百至上千）对内存扩展有挑战。

## 7. 优点

- **数据贡献**：首个大规模混合全景 RGB-D 数据集（10K 场景，299K 视点），包含真实 LiDAR 和合成光线追踪数据，填补训练数据空白。
- **创新方法**：可学习共视模块动态选择最优参考帧，避免启发式锚点漂移；全景几何分解策略降低监督难度并增强跨任务一致性。
- **工程技巧**：多 DPT 头在训练时强化学习，推理时可关闭点图头以零额外开销；固定全景投影消除内参歧义，实现直接度量预测。
- **充分评估**：在标准姿势/深度/点云指标上全面对比，并做详细消融，验证各组件有效性。
- **泛化展示**：展示了对不同风格、场景类型的稳健性。

## 8. 不足与局限

- **领域限制**：训练数据集中于室内结构化场景，零样本泛化到无界室外/航拍场景能力弱。
- **规模扩展**：当前模型对数百至上千全景视点的大规模重建受内存限制，未提供分治或可扩展方案（论文提到 scale-agnostic 重建为未来方向）。
- **对比方法**：仅对比了微调后的透视模型，缺少全景专用（如离线的 SfM 或全景 MVS）的基线，主要是传统方法太慢且效果差，但可能忽略一些现有全景深度估计方法的性能。
- **实验覆盖**：核心实验在合成子集上完成消融，真实子集上未做消融，可能存在域偏差。另外，单目深度零样本迁移实验未在正文中详细展开，需依赖补充材料。
- **潜在偏差**：数据集合成部分使用程序生成和资产库，可能不完全反映真实世界的极端光照、杂乱度或动态物体；真实子集采集设备为固定三脚架，与手持式全景采集仍存在差距。
