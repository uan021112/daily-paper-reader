## 摘要
大语言模型智能体越来越多地被用于以程序化方式将自然语言转化为3D场景，但现有系统专注于静态输出。仅从文本生成动态4D场景——其中液体流动、粒子发射、刚体级联、铰接机构运动——尽管作为可编辑内容以及作为视频生成和具身智能的物理基础训练数据具有价值，但仍未得到充分探索。两个挑战使动态情况与静态文本到场景的工作不同：智能体必须在一个连贯的场景中联合协调空间布局、多个物理求解器、时间序列、相机和光照，并且从渲染视频中验证运动正确性本质上比判断单张图像更难。我们提出SimWorlds：一个多智能体框架，能从文本生成动态、可编辑的4D场景，具有Blender特定的程序化知识，采用规划者-编码者-评审者工作流程驱动固定有序的构建阶段，通过确定性验证器强制执行分层场景协议，以及运行时状态检查工具套件，能捕获渲染图像无法揭示的机制故障。我们还引入了4DBuildBench，一个用于评估从文本提示生成的程序化动态3D场景的视觉保真度和物理一致性的基准。实验表明，SimWorlds优于先前的动态Blender生成基线。

## Abstract
LLM agents are increasingly used to translate natural language into 3D scenes in a procedural way, but existing systems focus on static output. Dynamic 4D scenes from text alone, in which liquids flow, particles emit, rigid bodies cascade, and articulated mechanisms move, remain largely unexplored despite their value as editable content and as physics-grounded training data for video generation and embodied AI. Two challenges set the dynamic case apart from static text-to-scene work: an agent must jointly coordinate spatial layout, multiple physics solvers, temporal sequencing, camera, and lighting in a single coherent scene, and verifying motion correctness from rendered video is fundamentally harder than judging a single image. We present SimWorlds: a multi-agent framework that produces dynamic, editable 4D scenes from text, with Blender-specific procedural knowledge, a planner-coder-reviewer workflow driving a fixed ordered sequence of construction stages, a layered scene protocol enforced by a deterministic verifier, and a runtime-state inspection tool suite that catches mechanism failures the rendered image cannot reveal. We also introduce 4DBuildBench, a benchmark for assessing both visual fidelity and physical consistency of the procedural dynamic 3D scenes generated from text prompts. Experiments show that SimWorlds outperforms prior dynamic Blender generation baselines.

- **挑战**：动态场景生成需要联合协调空间布局、多个物理求解器、时间序列、相机和光照，且从渲染视频验证运动正确性比判断单张图像难得多。现有系统仅靠渲染图像+视觉语言模型反馈，无法检测机制错误（如用关键帧动画冒充物理模拟）。
- **目标**：开发一个能生成可编辑、物理驱动的4D Blender场景系统，其动态效果由真实物理求解器实现，而非手动画关键帧伪装。

## 2. 方法论

### 核心思想
- **机制正确的程序合成**：将动态3D场景生成视为基于计划、机制感知的程序合成。生成过程不是一次性代码生成，而是分阶段构建，每个阶段经过确定性验证和视觉审查，确保机制正确性而非仅视觉合理性。

### 关键技术细节
1. **多智能体架构**（规划者-编码者-评审者）：
   - **规划者**（Strategic Planner）：一次性将用户文本提示转换为结构化场景计划（scene plan），包括对象、空间关系图、物理属性、运动阶段、渲染意图等。每个阶段开始时还会生成战术计划（tactical plan）。
   - **编码者**（Coder）：在每个阶段编写bpy Python脚本扩展场景状态。
   - **评审者**（Reviewer）：根据渲染预览和运行时状态读入，判断每个阶段的接受标准。
   - **规划者决策**：根据验证器和评审者反馈，决定前进（advance）、重试（retry）、重新计划（replan）或中止（abort）。

2. **分阶段构建管线**（Staged Construction Pipeline）：
   - 固定顺序的阶段：模型建模 → UV展开 → 纹理 → 变形设置 → 运动 → 相机 → 光照 → 渲染。
   - 每个阶段独立执行“编码-验证-评审”循环，失败则局部重试，避免早期错误污染后期。

3. **场景协议与确定性验证器**（Scene Protocol & Verifier）：
   - 在Blender原生场景图之上叠加三层层次结构（L1场景根、L2系统分组、L3逻辑对象）。
   - 通过自定义属性标记机制角色（rig/sim_cloth/sim_fluid/rigid等），定义表面接触、协同运动等关系。
   - 验证器包含四大类规则：结构规则（C1-C6, G1-G3）、几何规则（V1-V2，基于BVH距离测试）、软规则（W1-W5警告）、计划-状态匹配规则（R1-R18及T族规则）。可检测漂浮物体、未声明的穿透、缺失缓存等。

4. **工具套件**（Tool Suite）：
   - 提供状态观察工具（`blender_scene_state`返回修饰器栈、物理缓存、fcurve通道）、多粒度预览（`blender_single_mesh_preview`等）、运动审计工具（`blender_motion_sheet_preview`显示每帧的相位边界和烘焙状态）。
   - 通过MCP服务器暴露这些API给LLM智能体。

5. **知识库**（Knowledge Base）：
   - 自动从Blender官方API源（`bpy.types`类、`bpy.ops`操作符、手册）派生，按版本自动生成，无手动定制。智能体通过`blender_docs(query)`工具按需查询。

    - 确定性验证器检查协议规则 → 评审者判断感知标准。

## 3. 实验设计

### 数据集/基准
- **4DBuildBench**：作者自建基准，包含50个场景。
  - **机制轴**：5个Blender物理求解器类别（布料、流体、刚体、粒子、软体），每类9个提示；外加5个静态场景（家具室内场景）。
  - **难度轴**：每类动态提示分3个难度级别（D1单执行器 → D2同类复杂性 → D3跨类别交互），每级3个提示。
  - 共45个动态场景+5个静态场景。

- **编辑模式评估**：使用VIGA提出的**BlenderBench**（27个编辑任务，分3个难度等级：相机调整、多步属性编辑、组合编辑）。

### 对比方法
- **主要基线**：**VIGA**（Vision-as-Inverse-Graphics Agent），是唯一先前发表的具有专用动态场景模式的系统，也输出可编辑的Blender文件，且共享双智能体设计，可以隔离对比分阶段构建、场景协议和工具套件的贡献。

- 其他系统（LL3M、SceneCraft、Holodeck）因代码未放出或输出类型不符，未直接对比。

### 评估指标
1. **引擎状态审计**（Engine-state audit）：确定性检查，在无头Blender中读取运行时状态（修饰器、缓存、碰撞、约束、力场、关键帧密度），与每场景的手工标注的机制规格（42个类型谓词）对比。聚合指标：**MPR**（机制通过率，每个执行器的谓词平均分）和**SPR**（结构通过率，空间关系几何检验通过比例）。
2. **项目化VLM评判**（Itemized VLM judge）：使用GPT-5.5，接收提示和5个均匀采样帧，对每个命名对象、关系、动作进行二值化判定，输出5个维度分数（对象存在、空间关系、动作可见、视觉质量、美学），取平均作为VLM分数。

## 6. 主要结论与发现

1. **SimWorlds在4DBuildBench上全面优于VIGA**：
   - 总体MPR：0.87 vs 0.67（+0.20），SPR：0.89 vs 0.70（+0.19），VLM分数：0.82 vs 0.78（+0.04）。
   - 差距在复杂输入（多对象交互、长时间序列、跨类别求解器）上更明显。
2. **视觉验证的盲区**：VLM仅通过帧采样评估无法检测机制错误（如关键帧伪装），而引擎状态审计能捕捉。VIGA的视觉验证器因此接受伪造效果，SimWorlds的协议验证器则在构建阶段就捕获。
3. **编辑模式**：SimWorlds在BlenderBench上优于VIGA，在所有三个难度级别上PL、N-CLIP、VLM分数均更好。
4. **消融**：验证器和分阶段构建分别守护结构正确性和视觉抛光，彼此不可替代。

## 7. 优点

1. **明确区分视觉验证与机制验证**：首次将引擎状态审计（确定性、无LLM）引入LLM智能体的闭环，解决了“看起来对但机制错”的关键问题。
2. **分阶段构建管线**：根据Blender场景依赖关系固定阶段顺序，每个阶段独立验证和重试，错误局部化，避免累积。同时支持场景计划不变、检查点回滚等实用设计。
3. **轻量级场景协议**：在Blender原生集合和父级关系上叠加三层层次结构和关系图，使资产可验证、可组织，同时代码量小，不依赖额外存储。
4. **全面的工具套件**：让智能体能直接读取运行时状态（修饰器栈、物理缓存、fcurve通道等），而非仅看渲染图像，使评审者有更多信息。
5. **自导知识库**：自动从Blender官方源派生，不依赖手工整理，随版本更新。
6. **基准设计合理**：4DBuildBench沿机制轴和难度轴组织，每个场景有手工标注的机制规格，评估采用双轨制（引擎状态+ VLM），能区分“视觉正确但机制错误”的场景。
7. **在编辑任务上也表现良好**：无需架构修改，直接适用于BlenderBench，且优于VIGA。
