## 摘要
在这项工作中，我们重新审视了现代基于 Transformer 的前馈三维高斯泼溅（3DGS）预测方法中的几个关键设计选择。我们认为，将高斯均值回归为沿相机射线深度的常见做法是次优的，并提出仅使用自监督渲染损失来直接回归三维均值坐标。这种方案使我们能够从标准的仅编码器设计转向具有可学习高斯 Token 的编码器-解码器架构，从而将预测基元的数量与输入图像分辨率和视图数量解耦。我们提出的方法 TokenGS 展示了对位姿噪声和多视图不一致性更强的鲁棒性，同时自然地支持在 Token 空间中进行高效的测试时优化，且不会损害已学习的先验。TokenGS 在静态和动态场景中均实现了最先进的前馈重建性能，产生了更具正则性的几何结构和更均衡的 3DGS 分布，同时无缝地恢复了诸如静动态分解和场景流等涌现的场景属性。

## Abstract
In this work, we revisit several key design choices of modern Transformer-based approaches for feed-forward 3D Gaussian Splatting (3DGS) prediction. We argue that the common practice of regressing Gaussian means as depths along camera rays is suboptimal, and instead propose to directly regress 3D mean coordinates using only a self-supervised rendering loss. This formulation allows us to move from the standard encoder-only design to an encoder-decoder architecture with learnable Gaussian tokens, thereby unbinding the number of predicted primitives from input image resolution and number of views. Our resulting method, TokenGS, demonstrates improved robustness to pose noise and multiview inconsistencies, while naturally supporting efficient test-time optimization in token space without degrading learned priors. TokenGS achieves state-of-the-art feed-forward reconstruction performance on both static and dynamic scenes, producing more regularized geometry and more balanced 3DGS distribution, while seamlessly recovering emergent scene attributes such as static-dynamic decomposition and scene flow.

### 1. 核心问题与研究动机
*   **核心问题**：当前主流的前馈 3DGS 重建方法（如 GS-LRM）多采用“仅编码器（Encoder-only）”架构，将高斯均值预测为沿相机射线的深度值。
*   **研究动机**：
    *   **耦合限制**：高斯基元的数量被绑定在输入图像的分辨率和视图数量上（如每个像素一个点），导致冗余度高且计算量随输入线性增长。
    *   **鲁棒性差**：依赖射线深度预测使得模型对相机位姿噪声极其敏感，且容易产生“尖刺”状的几何畸变。
    *   **优化困难**：直接在测试时优化高斯参数容易破坏模型学习到的全局先验，导致在稀疏视图下过拟合。

### 2. 方法论
*   **核心思想**：引入**可学习的高斯 Token**，采用**编码器-解码器（Encoder-Decoder）**架构，将 3D 高斯的预测与输入像素彻底解耦。
*   **关键技术细节**：
    *   **直接 3D 坐标回归**：不再预测深度，而是直接回归高斯中心在全局坐标系下的 $(x, y, z)$ 坐标。
    *   **编码器-解码器架构**：编码器提取多视图图像特征；解码器利用一组固定数量的可学习 Token 作为 Query，通过交叉注意力机制从图像特征中提取信息，并独立回归高斯属性。
    *   **可见性损失（Visibility Loss）**：为解决直接回归坐标时，视锥外的点无梯度的问题，提出一种正则化项，强制高斯中心投影在至少一个监督视图的图像平面内。
    *   **动态场景建模**：将 Token 分为静态和动态两组，动态 Token 加入时间嵌入，并使用因果掩码（Causal Mask）使动态部分依赖于静态结构，从而实现动静分解。
    *   **测试时缩放（Test-Time Scaling）**：
        *   **上下文扩展**：推理时可输入比训练时更多的视图，而高斯点数保持不变。
        *   **Token 微调（Token Tuning）**：仅优化 Token 的嵌入向量而非网络权重或高斯参数，在提升质量的同时保留模型先验。

### 3. 实验设计
*   **数据集**：
    *   **静态场景**：RealEstate10K (RE10K) 和 DL3DV-10K。
    *   **动态场景**：Kubric 4D 数据集。
    *   **物体级测试**：Objaverse（用于初步验证）。
*   **对比基准（Baselines）**：MVSplat, DepthSplat, GS-LRM, BTimer 等前馈重建模型。
*   **评估指标**：PSNR, SSIM, LPIPS，以及几何质量的定性可视化。

### 4. 资源与算力
*   **硬件**：使用了 **8 台 NVIDIA A100 GPU**。
*   **训练时长**：

*   **推理效率**：在 DL3DV 上的推理时间约为 0.05s - 0.13s（不含 Token Tuning）。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了从 2 视图到多视图的静态重建、视图外推（Extrapolation）、相机位姿噪声鲁棒性测试、动态场景重建及场景流（Scene Flow）恢复。
*   **消融实验**：详细分析了可见性损失的作用、Token 数量的影响、以及 Token Tuning 与直接优化高斯参数的对比。
*   **评价**：实验设计全面，对比了最新的 SOTA 方法，且在不同任务（静态 vs 动态）和不同挑战（噪声 vs 外推）下均有验证，客观性较强。

### 6. 主要结论与发现
*   **解耦优势**：TokenGS 能够以更少的高斯点数（减少约 50%-70%）达到甚至超过像素对齐方法的重建质量。
*   **几何规整**：直接回归 3D 坐标消除了深度预测带来的尖刺伪影，生成的几何结构更加平滑、完整。
*   **鲁棒性提升**：在相机位姿存在 1-10 度旋转噪声时，TokenGS 的性能下降远小于 GS-LRM。
*   **涌现属性**：模型在动态场景中自然地学会了动静分解，并能提取出一致的场景流轨迹。

### 7. 优点
*   **架构创新**：首次在前馈 3DGS 中成功应用编码器-解码器架构，实现了点数与分辨率的解耦。
*   **灵活性高**：支持测试时通过增加视图或微调 Token 来提升质量，具有良好的可扩展性。
*   **多功能性**：一个框架同时解决了静态重建、动态建模和视图外推问题。

### 8. 不足与局限
*   **精细度限制**：虽然几何更规整，但在处理极高频的纹理细节时，可能仍受限于 Token 的表示容量。
*   **测试时优化成本**：Token Tuning 虽然有效，但需要额外的迭代步骤（如 50 步），牺牲了部分前馈模型的速度优势。
*   **大规模场景**：对于超大规模的室外场景，固定数量的 Token 可能会面临容量瓶颈。
