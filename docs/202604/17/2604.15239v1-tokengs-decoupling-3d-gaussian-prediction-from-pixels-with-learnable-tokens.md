## 摘要
在这项工作中，我们重新审视了用于前向 3D 高斯泼溅（3DGS）预测的现代基于 Transformer 方法的几个关键设计选择。我们认为，将高斯均值回归为沿相机射线的深度的常用做法并非最优，并提出仅使用自监督渲染损失直接回归 3D 均值坐标。这种表述使我们能够从标准的仅编码器设计转向具有可学习高斯 Token 的编码器-解码器架构，从而将预测基元的数量与输入图像分辨率和视图数量解耦。我们提出的方法 TokenGS 展示了对位姿噪声和多视图不一致性更强的鲁棒性，同时自然地支持在 Token 空间中进行高效的测试时优化，且不会降低已学习的先验。TokenGS 在静态和动态场景中均实现了最先进的前向重建性能，产生了更具正则性的几何结构和更平衡的 3DGS 分布，同时无缝恢复了诸如动静分解和场景流等涌现的场景属性。

## Abstract
In this work, we revisit several key design choices of modern Transformer-based approaches for feed-forward 3D Gaussian Splatting (3DGS) prediction. We argue that the common practice of regressing Gaussian means as depths along camera rays is suboptimal, and instead propose to directly regress 3D mean coordinates using only a self-supervised rendering loss. This formulation allows us to move from the standard encoder-only design to an encoder-decoder architecture with learnable Gaussian tokens, thereby unbinding the number of predicted primitives from input image resolution and number of views. Our resulting method, TokenGS, demonstrates improved robustness to pose noise and multiview inconsistencies, while naturally supporting efficient test-time optimization in token space without degrading learned priors. TokenGS achieves state-of-the-art feed-forward reconstruction performance on both static and dynamic scenes, producing more regularized geometry and more balanced 3DGS distribution, while seamlessly recovering emergent scene attributes such as static-dynamic decomposition and scene flow.

### 1. 核心问题与研究动机
*   **核心问题**：当前主流的前馈 3DGS 预测模型（如 GS-LRM）通常采用“仅编码器（Encoder-only）”架构，将高斯均值预测为沿相机射线的深度值（像素对齐）。
*   **研究动机**：
    *   **耦合限制**：高斯基元的数量被绑定在输入图像的分辨率和视图数量上，导致冗余度高且缺乏灵活性。
    *   **鲁棒性差**：依赖射线深度预测使得模型对相机位姿噪声和多视图不一致性非常敏感。
    *   **优化退化**：在测试时直接优化高斯参数（Test-time Optimization）往往会破坏模型学习到的全局先验，导致几何结构畸变。
    *   **动态挑战**：在动态场景中，难以通过简单的像素变形来处理复杂的物体运动。

### 2. 方法论
*   **核心思想**：提出 **TokenGS** 框架，通过**编码器-解码器（Encoder-Decoder）**架构和**可学习的高斯 Token**，将 3D 高斯的预测与输入像素彻底解耦。
*   **关键技术细节**：
    *   **直接 3D 坐标回归**：不再预测深度，而是直接在全局坐标系中回归高斯的 3D 中心坐标 $(x, y, z)$。
    *   **高斯 Token 解码器**：使用一组可学习的 Token 作为 Query，通过交叉注意力（Cross-Attention）机制从图像特征中提取信息。每个 Token 独立解码出一组高斯属性，使高斯数量成为可控的超参数。
    *   **可见性损失（Visibility Loss）**：为解决直接回归坐标导致的“零梯度”问题（高斯点落在视锥外无监督），引入 $L_{vis}$ 约束，将不在任何视图内的点拉回可见区域。
    *   **动静分解建模**：将 Token 分为静态和动态两组。动态 Token 加入时间嵌入，并采用因果注意力掩码（只能观察静态 Token），从而实现场景流（Scene Flow）的自动恢复。
    *   **测试时 Token 微调（TT）**：在推理阶段，仅优化 Token 的 Embedding 而非网络权重或高斯参数，从而在提升质量的同时保留模型先验。

*   **消融实验**：详细探讨了可见性损失的作用、Token 数量的影响、以及 Token 微调与直接参数微调的对比。

### 6. 主要结论与发现
*   **性能领先**：TokenGS 在多个数据集上达到了 SOTA 水平，且使用更少的高斯基元（如在 DL3DV 上减少 72% 的高斯量）仍能保持竞争力。
*   **鲁棒性增强**：在相机位姿存在 1-10 度噪声的情况下，TokenGS 的性能下降远小于像素对齐的方法。
*   **几何质量**：直接回归 3D 坐标消除了深度预测中常见的“尖刺”伪影，生成的几何结构更平滑、更完整。
*   **涌现属性**：模型能够自动学习到动静分离，并能通过动态 Token 的轨迹提取出一致的场景流。

### 7. 优点
*   **架构创新**：成功将 DETR 式的 Token 机制引入 3DGS 重建，实现了预测规模与输入分辨率的解耦。
*   **测试时缩放**：Token 微调（TT）提供了一种优雅的测试时优化方案，避免了过拟合导致的几何崩溃。
*   **多功能性**：一个模型架构同时解决了静态重建、动态追踪和视角外推等多种任务。

### 8. 不足与局限
*   **计算开销**：虽然前馈预测很快，但测试时 Token 微调（TT）需要数十次迭代，增加了实际部署的时间成本。
*   **精细度限制**：与所有前馈模型类似，在处理极大规模环境或极其微小的几何细节时，仍与耗时的单场景优化方法存在差距。
*   **初始化敏感**：直接回归 3D 坐标对初始化和可见性损失的依赖较强，若参数设置不当可能导致训练不稳定。
